# TaskK 入口与演示（集成）规格

最后一个 Task：把编辑器接入游戏入口，提供演示图，清理 MDK 模板残留，端到端打通。

## 目标
1. **物品开屏**：右键 `nodegraph:graph_editor` 物品 → 打开 `NodeGraphScreen`（含演示图）。
2. **演示图**：`DemoGraphFactory` 构造一个展示性 `NodeGraph`（类型配色 / 连线 / 自动转换警告 / 节点组 / 可被求值引擎求值）。
3. **清理 MDK**：删除 `Config.java`；主类移除 `EXAMPLE_BLOCK` / `EXAMPLE_BLOCK_ITEM` / `EXAMPLE_ITEM` / `EXAMPLE_TAB` / `BLOCKS` / `CREATIVE_MODE_TABS` / MDK 日志钩子。
4. **端到端**：`compileJava` 通过；视觉与交互由用户 `runClient` 确认（本环境无 runClient 实测条件）。

## 前置
- TaskA~J 全部完成并提交。
- MC 1.20.1 客户端 API（已从 sources jar 核实）：
  - `net.minecraft.world.item.Item`：`public InteractionResultHolder<ItemStack> use(Level level, Player player, InteractionHand usedHand)`；默认对非食物返回 `pass`。
  - `net.minecraft.world.InteractionResultHolder`：`<T> sidedSuccess(T object, boolean isClientSide)`（客户端 SUCCESS 停止后续、服务端 PASS）。
  - `net.minecraft.client.Minecraft`：`public void setScreen(@Nullable Screen)`。
  - `net.minecraftforge.api.distmarker.{Dist, OnlyIn}`（DistExecutor 在本 sources jar 不存在 → 不使用）。
- 复用 TaskE `NodeGraphScreen(Component title, NodeGraph graph)`、TaskB `Evaluator`。

## 关键设计决策

### D1 物品开屏模式（自定义 Item + isClientSide 守卫 + @OnlyIn helper）
自定义 `GraphEditorItem extends Item`，override `use`：
```
if (level.isClientSide()) {
    openEditor();   // @OnlyIn(Dist.CLIENT) 静态 helper
}
return InteractionResultHolder.sidedSuccess(stack, level.isClientSide());
```
`openEditor()` 标注 `@OnlyIn(Dist.CLIENT)`，内含 `Minecraft.getInstance().setScreen(new NodeGraphScreen(...))`。
- 专用服务器安全：`isClientSide` 守卫保证专用端不进入该分支；`@OnlyIn` helper 在专用端被 Forge ClassTransformer 剥离，即使误调用也无客户端类链接（方法体引用 `Minecraft`/`NodeGraphScreen` 惰性加载）。
- 不使用 `PlayerInteractEvent.RightClickItem`（仅 air 右键触发，覆盖不全）；不使用 `DistExecutor`（sources jar 未提供）。

### D2 演示图 DemoGraphFactory（纯数据构造，可单测）
`com.nodegraph.nodegraph.client.DemoGraphFactory.create() -> NodeGraph`（返回 `api.model.NodeGraph`）。**不加 @OnlyIn**（仅用 Component/ResourceLocation/类型/定义，公共侧可达，可 JUnit）。内容：
- 类型（配色）：`number`（0xFF6ACCD9 青蓝）、`byte`（0xFFE6C07B 黄棕）。
- 转换规则：`number → byte`（`v -> Byte.valueOf(((Number)v).byteValue())`）。
- 节点定义（3 种）：
  - `constant_number`：1 widget `value`(TEXT, 默认 "0")，1 输出 `out`(number)；function：`{ out: Long.parseLong((String) widgetValues.getOrDefault("value","0")) }`。
  - `add`：2 输入 `a`/`b`(number)，1 输出 `result`(number)；function：`{ result: ((Number)inputValues.get("a")).longValue() + ((Number)inputValues.get("b")).longValue() }`。
  - `to_byte`：1 输入 `in`(byte)，1 输出 `out`(byte)；function：`{ out: ((Number)inputValues.get("in")).byteValue() }`。
- 节点实例：`constA`(constant_number, value="10") @ (40,40)；`constB`(constant_number, value="32") @ (40,160)；`sum`(add) @ (320,100)；`clamp`(to_byte) @ (620,100)。
- 连线：`constA.out → sum.a`、`constB.out → sum.b`、`sum.result → clamp.in`（**自动转换 number→byte，橙色警告**）。
- 组：`createGroup("Inputs", 20, 20, 200, 260)` 包围 constA+constB（setNodeGroup）。

### D3 求值演示（端到端引擎验证）
`openEditor()` 在构造 screen 前用 `Evaluator().evaluateAll(graph)` 求值演示图，`LOGGER.info` 输出 `clamp.out` 结果（预期 `(byte)42`），证明 TaskB 引擎端到端可用。

### D4 清理 MDK
- 删 `Config.java`（整文件）。
- 主类 `NodeGraph.java` 移除：`BLOCKS` / `CREATIVE_MODE_TABS` DeferredRegister、`EXAMPLE_BLOCK` / `EXAMPLE_BLOCK_ITEM` / `EXAMPLE_ITEM` / `EXAMPLE_TAB`、`commonSetup` / `addCreative`(改写) / `onServerStarting`、`ClientModEvents.onClientSetup` 的 MDK 日志、`ModLoadingContext.registerConfig`。

### D5 资源文件

## 不变量
- `MODID="nodegraph"` 不变。
- `ITEMS` DeferredRegister 保留并注册到 modEventBus。
- 专用服务器加载主类与 GraphEditorItem 时不触发客户端类加载（isClientSide + @OnlyIn）。
- DemoGraphFactory 返回的图可被 `Evaluator.evaluateAll` 求值且 `clamp.out == (byte)42`。

## 异常
- `Long.parseLong` 对非法 widget 值抛 `NumberFormatException`（演示图默认值合法；非演示输入由 function 自负）。
- 求值遇环抛 `CycleException`（演示图无环）。

## 副作用
- 右键物品：客户端打开 Screen（setScreen），服务端无状态变化。
- `openEditor` 每次右键新建一个 `NodeGraph`（演示图不持久化）。
