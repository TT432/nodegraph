# TaskK 反馈问题

## 1. 规格 D4 的 CreativeModeTab 常量名错误
- 规格文档 D4 写"监听 CreativeModeTabs.TOOLS accept(GRAPH_EDITOR_ITEM)"。
- 实际 MC 1.20.1（forge 47.4.20 sources 核实）`CreativeModeTabs` 无 `TOOLS` 字段；正确常量为 **`TOOLS_AND_UTILITIES`**（类型 `ResourceKey<CreativeModeTab>`，`createKey("tools_and_utilities")`，标题 key `itemGroup.tools`）。
- 实现已用 `CreativeModeTabs.TOOLS_AND_UTILITIES`，非设计缺陷，纯属规格 API 名笔误。
- 其他备选常量：`BUILDING_BLOCKS`/`COMBAT`/`FOOD_AND_DRINK`/`NATURAL`/`FUNCTIONAL`/`REDSTONE`/`OP_BLOCKS`/`SPAWN_EGGS`/`INGREDIENTS`/`SEARCH` 等（均为 `ResourceKey<CreativeModeTab>`）。

## 2. ResourceLocation(String,String) removal 警告（沿用前序 Task）
- DemoGraphFactory/GraphEditorItem/TestDemoGraph 均用 `new ResourceLocation("nodegraph", id)`，触发 removal 警告。
- 已知良性，TaskA 起即存在，与本映射版本兼容（无替代 API 可用）。

## 3. FMLJavaModLoadingContext.get() removal 警告
- 主类构造沿用 MDK 模板的 `FMLJavaModLoadingContext.get().getModEventBus()`，触发 removal 警告。
- 1.20.1 仍是标准写法，迁移到新 API（如 ModContainer）属重构范畴，不在本 Task 范围。
