# TaskM 规格说明：实时求值回显 + Widget 编辑

## 背景
用户在 runClient 测试中发现：
1. constant 节点的 InputWidget（TEXT）不可编辑——TaskF 把 widget 渲染为纯文本，交互留待后续，本 Task 补齐。
2. ADD 等有结果的节点应展示中间结果——当前 Evaluator 仅在开屏时手动调用一次（日志），无实时回显。

## 目标
- **M1 实时求值回显**：渲染时对图求值，在每个 output 端口标签处显示当前值（`name = value`）。求值失败（环/异常）优雅降级，不崩溃、在该节点显示错误标记。
- **M2 TEXT widget 编辑**：点击 TEXT 类型的 InputWidget 行进入编辑模式（EditBox），Enter 确认（入 undo 栈），ESC 取消。SLIDER/BUTTON_GROUP/DROPDOWN 不在本 Task 范围（留作后续）。

## 前置（复用）
- TaskB `Evaluator.evaluateAll(NodeGraph)->EvaluationResult`；`EvaluationResult.outputsOf(NodeId)->Map<String,Object>`（未参与/异常→IllegalArgumentException）；`CycleException`/`EvaluationException`。
- TaskC `SetWidgetValueCommand(graph, NodeId, String widgetKey, Object newValue)`。
- TaskF `NodeLayout.inputWidget(i)->Rect`/`pickInputWidget(wx,wy)->Optional<Integer>`；`NodeRenderer.render`（本 Task 改签名）。
- MC 1.20.1 `EditBox(font,x,y,w,h,Component)`：setValue/getValue/keyPressed/charTyped/mouseClicked/render/setFocused/setCanLoseFocus/setHint（已于 AddNodeOverlay 验证）。

## 关键设计决策
- **D1 每帧求值**：renderWidget 开头 `new Evaluator().evaluateAll(graph)`，try/catch RuntimeException→result=null。图小（演示级）开销可忽略；dirty 缓存优化留作后续打磨（不在本 Task）。
- **D2 值显示位置**：output 端口标签内联。原标签 `name` 右对齐到端口点左侧；有值时改为 `name = value`（同样右对齐）。值用 `String.valueOf`；null 值不显示。不新增行、不改布局。
- **D3 错误标记**：result!=null 但某节点 outputsOf 抛异常（环节点/求值异常节点）→ 该节点 output 标签显示 `name = !`（红色 `0xFFFF5555`），提示异常但不中断渲染。
- **D4 编辑态模态**：开启 EditBox 后，键盘优先路由 EditBox；点击 EditBox 内=转发，点击外=确认+关闭并继续处理本次点击。一次只编辑一个 widget。
- **D5 EditBox 坐标=屏幕坐标**（不进 PoseStack scale）：widget 行世界矩形经 worldToScreen 得屏幕矩形，EditBox 用该矩形（1:1 字号）。编辑期间 NodeRenderer 跳过该行文本（传 editingWidgetIndex），由 EditBox 接管渲染。
- **D6 确认=SetWidgetValueCommand**：值以 String 形式存（TEXT widget currentValue 是 String）。若新值与原值相等→不 apply（避免空 undo 条目）。ESC 取消→不 apply。
- **D7 编辑触发**：mouseClicked(button0) 路由顺序——activeEdit 失焦判定 → addNodeOverlay → menu → controller(头/端口) → **widget pick(TEXT)** → box select。命中非 TEXT widget 当前不处理（无操作，继续 box select）。
- **D8 求值不阻塞渲染**：任何异常（含 OOM 之外的 RuntimeException）都 catch，保证渲染循环稳定。

## 交付物
- `NodeRenderer.render` 签名扩展：新增参数 `Map<String,Object> outputs`（可 null）+ `int editingWidgetIndex`（-1=无）。output 标签据 outputs 显示 name[ = value][ = !]；editingWidgetIndex 行跳过文本。
- `NodeGraphWidget`：字段 `activeEdit`/`editNode`/`editWidgetIndex`/`editKey`；mouseClicked 加 activeEdit 失焦 + widget pick；renderWidget 开头求值+传参+末尾画 EditBox；editKey/editChar/openEdit/confirmEdit/cancelEdit。
- `NodeGraphScreen.keyPressed`：首行优先 `canvas.editKey(keyCode)`。

## 不变量
- 求值异常绝不抛到渲染循环外。
- EditBox 关闭后 activeEdit=null，不影响后续事件。
- editingWidgetIndex 仅对单个节点生效（editNode）。

## 异常
- 求值抛 RuntimeException → result=null（所有节点不显示值）。
- outputsOf 单节点抛 IllegalArgumentException → 该节点显示 `!`。

## 副作用
- 每帧重新求值（纯读，不改 graph）。
- 确认编辑 apply SetWidgetValueCommand（改 graph + 入 undo 栈）。

## 验证
- compileJava+compileTestJava 通过；gradlew test 全过（无新增测试——客户端渲染层无单测条件；逻辑层未改）。
- runClient：演示图 ADD 节点 result 标签显示 `result = 42.0`；constant.out 显示 `out = 10.0`；to_byte.out 显示 `out = 42`。点击 constant 的 value 行→EditBox 出现→改值回车→节点移动/连线不变，值更新，下游 result 重算。
