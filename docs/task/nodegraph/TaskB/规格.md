# TaskB 求值引擎

## 目标
实现 Blender 风格的惰性求值引擎：对节点图求值，递归解析每个输入端口的来源（连线源输出，必要时自动转换），调用节点 `NodeFunction` 得到输出。纯 Java 层，仅依赖 TaskA 的数据模型（NodeGraph / Node / Connection / NodeDefinition / NodeFunction / TypeConversionRule），不依赖 MC 客户端渲染类。可用 JUnit 测试。

## 前置条件
- TaskA 已完成（commit 45a254a）。复用：
  - `NodeGraph`：`node(id)`、`nodes()`、`inputConnection(node,in)->Optional<Connection>`、节点 `inputs()/outputs()/widgets()/definition()`。
  - `Connection`：`fromNode()/fromOutput()/toNode()/toInput()/isAutoConverted()/rule()`。
  - `NodeFunction.evaluate(Map<String,Object> inputValues, Map<String,Object> widgetValues) -> Map<String,Object>`（输入端口键→值；widget 键→值；返回输出端口键→值）。
  - `TypeConversionRule.apply(Object)->Object`。

## 范围与交付物
包 `com.nodegraph.nodegraph.api.eval`：

### Evaluator（无状态，可复用，线程安全）
- `EvaluationResult evaluateAll(NodeGraph graph)`：对图中**每个**节点求值（共享一个运行内缓存，菱形/扇出依赖每节点只算一次）。
- `Map<String,Object> evaluate(NodeGraph graph, NodeId node)`：惰性求值单节点（及其可到达的全部依赖），返回该节点输出 map。

### EvaluationResult
- `Map<NodeId,Map<String,Object>> outputs()`：本次运行所有已求值节点的输出。
- `Map<String,Object> outputsOf(NodeId node)`：便捷查询；节点未参与本次求值 → 抛 `IllegalArgumentException`。
- `Set<NodeId> evaluatedNodes()`：本次实际求值过的节点集合（不含仅被读取缓存的）。

### CycleException extends RuntimeException
- `List<NodeId> cycle()`：构成环的节点序列（按运行栈顺序，首尾为同一节点即环的闭合点）。

### EvaluationException extends RuntimeException
- `NodeId node()`：求值失败的节点。
- `Throwable cause()`：来自 `NodeFunction` 的原始异常。

### EvaluationContext（包级私有，内部状态）
- 持有 `Map<NodeId,Map<String,Object>> done`（缓存）与 `Deque<NodeId> stack` + `Set<NodeId> onStack`（环检测）。
- 每次求值运行新建一个实例；不跨运行复用。

## 求值算法（单节点）
```
evalNode(N):
  if N in done: return done[N]
  if N in onStack: throw CycleException(stack 自 N 首次出现处到栈顶 + N)
  stack.push(N); onStack.add(N)
  inputValues := {}
  for each input port index i of N:
    c := graph.inputConnection(N, i)
    if c 存在:
      srcOuts := evalNode(c.fromNode())               // 递归
      outKey  := N'.outputs()[c.fromOutput()].key()    // N' = c.fromNode 的节点
      v := srcOuts.get(outKey)
      if c.isAutoConverted() and c.rule() != null:
        v := c.rule().apply(v)
      inputValues[N.inputs()[i].key()] := v
    else:
      inputValues[N.inputs()[i].key()] := null         // 未连接：键存在，值 null
  widgetValues := { w.key() : w.currentValue() for w in N.widgets() }
  try:
    outs := N.definition().function().evaluate(inputValues, widgetValues)
  catch Throwable t:
    throw EvaluationException(N, t)
  done[N] := outs
  stack.pop(); onStack.remove(N)
  return outs
```

## 不变量
1. 单次求值运行内，每个节点至多调用其 `NodeFunction` 一次（缓存生效）。
2. `inputValues` 的键集 = 节点 definition 的全部 input port key；未连接端口值 = `null`。function 可据此回退到 widgetValues。
3. 引擎**透传** function 返回的 map：不补默认值、不裁剪多余键。function 作者负责为每个 output port 提供值。
4. 自动转换仅在 `connection.isAutoConverted() && connection.rule()!=null` 时应用。
5. 环以"运行栈灰点"检测：报告从该节点首次入栈到再次命中之间的栈片段。
6. Evaluator 无状态；求值过程不修改 NodeGraph（仅读）。

## 异常行为
- 求值遇环 → `CycleException`（含环节点列表）。
- `NodeFunction` 抛任意 `Throwable` → 包装为 `EvaluationException`（含 NodeId + cause）。
- `evaluate(graph, node)` 未知 NodeId → `IllegalArgumentException`（透传自 `NodeGraph.node(id)`）。
- Connection 的 `fromOutput` 索引对源节点 outputs 越界 → `IllegalStateException`（数据损坏；NodeGraph 正常不会产生）。

## 副作用
- 无 IO、无持久化、不修改 NodeGraph。
- Evaluator 实例无可变状态。

## 决策记录
- **未连接输入保留键(null)**：让 function 看到完整输入 schema，自行决定回退策略（如回退到 widgetValues）。比"省略键"更显式，且与 function 作者预期的端口键集一致。
- **转换规则取自 connection.rule()**（connect 时固化），求值器不查 TypeRegistry：使求值器只依赖 NodeGraph，且转换语义与连线时刻一致（即使之后 TypeRegistry 变更也不影响既有连线）。
- **缓存为运行局部**（EvaluationContext），不跨运行：避免脏数据；与"惰性 + 单 pass"目标一致。
- **环以运行栈检测**而非拓扑预检：惰性求值只需检测目标可达路径上的环；预检需全图扫描，浪费且语义不必要。
- **透传 function 输出 map**：引擎不臆断默认值；function 契约由作者保证。
- 提供 `evaluateAll` 与单节点 `evaluate` 两个入口：前者用于"刷新整图"，后者用于"按需取值"。共用 EvaluationContext。

## 验证（JUnit 5）
`src/test/java/com/nodegraph/nodegraph/api/eval/TestEvaluator.java`：
1. **线性链** A→B→C（每节点 1 in 1 out，function 透传+1）：C 输出 = A 初值变换链。
2. **菱形（缓存）** A→B, A→C, B→D, C→D：A 只求值一次（用计数 function 验证）。
3. **未连接输入**：inputValues 含该键且值为 null。
4. **widget 透传**：节点有 widget，function 返回 widget 值。
5. **自动转换**：int→byte 规则，连线后求值应用转换。
6. **环检测**：A→A 自环，或 A→B→A 二元环 → `CycleException` 且 `cycle()` 非空。
7. **单节点 evaluate**：只求值目标及其依赖。
8. **function 异常包装**：function 抛 `RuntimeException` → `EvaluationException`，`node()` 正确，`cause` 一致。
9. **evaluateAll**：所有节点均在 `evaluatedNodes()` / `outputs()` 中。
