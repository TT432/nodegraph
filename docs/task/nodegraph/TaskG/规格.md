# TaskG 连线渲染（客户端）

## 目标
在 `NodeGraphWidget` 画布上渲染节点间的连线（贝塞尔曲线）、按类型配色、标记自动转换警告、并预览线渲染能力（供 TaskH 拖拽连线时使用）。复用 TaskE（Viewport）+ TaskA（Connection）+ TaskF（NodeLayout 端口锚点）。

## 前置
- TaskA：`Connection`(fromNode/fromOutput/toNode/toInput/isAutoConverted)、`NodeGraph`(connections/node)、`Port`(value()->TypedValue)、`Type`(color())、`Node`。
- TaskE：`Viewport`(worldToScreenX/Y/scale)、`NodeGraphWidget`(renderWidget 钩子、getX/getY/width/height)。
- TaskF：`NodeLayout`(outputPort(i)/inputPort(i) -> PortAnchor(x,y,index)、NODE_WIDTH)。
- MC 1.20.1：`GuiGraphics.fill(int,int,int,int,int)`（已核实，唯一可用的"画线"等价原语，轴对齐矩形）；无任意角度画线 API（`hLine`/`vLine` 本质是 fill；LINES/LINE_STRIP 模式需 POSITION_COLOR_NORMAL + shader，复杂且本环境无法 runClient 验证）。

## 关键设计决策

### D1 连线在节点之下渲染（顺序）
`renderWidget` 流程：背景 fill → enableScissor → renderGrid → **renderConnections（新）** → renderNodes → disableScissor → tooltip。节点端口方块（TaskF）盖住连线起止点，视觉干净。

### D2 屏幕坐标 + 贝塞尔采样 + 段包围盒 fill
- 锚点经 `viewport.worldToScreen` 转为屏幕坐标。
- 三次贝塞尔 P0(输出锚点) → P3(输入锚点)，控制点 P1=P0+(dx,0)、P2=P3-(dx,0)，dx=max(|P3.x-P0.x|*0.5, MIN_CURVE_DX)。控制点水平偏移使曲线在两端切线水平，符合"左出右入"视觉惯例。
- 采样 N 段（按屏幕弧长自适应：N = clamp(round(屏幕距离/STEP), MIN_SEGMENTS, MAX_SEGMENTS)），得屏幕点序列 s0..sN。
- 对每对相邻 (si, si+1)：画其轴对齐包围盒（向外膨胀 HALF_THICKNESS）的 fill，使相邻段重叠区域自然填充 → 无缝连续曲线。此为 `hLine`/`vLine` 思路的泛化，斜段呈阶梯但步长小（STEP=2px）时阶梯不可见。
- 屏幕坐标绘制 → 粗细恒定（不随缩放变化），视觉一致。

### D3 配色规则
- **正常连线**：颜色取源输出端口的 `Type.color()`（fromNode.outputs()[fromOutput].value().type().color()）。与 TaskF 端口方块同色，端到端一致。
- **自动转换连线**（connection.isAutoConverted()）：用警告色 `WARN_COLOR=0xFFFFAA00`（橙），覆盖类型色，提示"已自动转换"。
- **预览线**（拖拽中）：半透明（alpha=0x80）类型色；兼容性目标类型未知时用白色半透明。

### D4 预览线渲染能力（数据源归 TaskH）
- `NodeGraphWidget` 新增 `ConnectionDrag` 状态字段（package-private static 嵌套类，含 fromLayout+outIdx 或 from 锚点 + 当前鼠标世界坐标 + 方向 + 颜色）。默认 null。
- TaskG 交付：渲染方法 `ConnectionRenderer.renderPreview`；widget 在 renderConnections 之后渲染 pending（pending!=null 时）。TaskH 负责在拖拽起止时设置/清空 pending。
- 预览线为直线 + 贝塞尔混合：起点=源端口锚点，终点=当前鼠标世界坐标，控制点同 D2 规则。

### D5 常量
- `STEP=2.0`（屏幕像素采样步长）
- `MIN_SEGMENTS=12`, `MAX_SEGMENTS=64`
- `MIN_CURVE_DX=24.0`（世界单位，最小水平控制偏移，防止近距离节点曲线过平）
- `THICKNESS=2`（屏幕像素，连线粗细），`HALF_THICKNESS=1`
- `PREVIEW_THICKNESS=1`
- `WARN_COLOR=0xFFFFAA00`（自动转换警告，橙）
- `PREVIEW_ALPHA=0x80`（预览线半透明，作用于 ARGB 高字节）

### D6 自动转换警告的额外标记
除橙色外，自动转换连线沿曲线中点叠加一个小的警告方块标记（`WARN_MARK_SIZE=4` 屏幕像素，`WARN_COLOR`），使警告在橙色与某些类型色相近时仍可辨识。标记为中点 fill 正方形。

## 交付物

### 1. `com.nodegraph.nodegraph.client.render.ConnectionRenderer`（final，工具类 private 构造）
- `static void render(GuiGraphics g, Viewport vp, int originX, int originY, NodeLayout from, int outIdx, NodeLayout to, int inIdx, int color, int thickness)`：贝塞尔曲线渲染（D2）。内部：取 outputPort/outPort 锚点→世界坐标→屏幕坐标→computeControlPoints→sample→逐段 fill 包围盒。
- `static void renderPreview(GuiGraphics g, Viewport vp, int originX, int originY, double fromWx, double fromWy, double toWx, double toWy, int color)`：预览贝塞尔（D4），thickness=PREVIEW_THICKNESS，alpha 处理由调用方颜色值承载。
- `static void renderWarnMark(GuiGraphics g, double sx, double sy)`：中点警告方块。
- 私有 `static double[] cubic(double[] p0,p1,p2,p3, double t)`、`static List<int[]> sampleCurve(...)`、`static void fillSegment(GuiGraphics, double x0,double y0,double x1,double y1,int halfThickness,int color)`（段包围盒 fill）。
- 暴露常量供 widget 用：`WARN_COLOR`、`WARN_MARK_SIZE`、`STEP` 等（public static final）。

### 2. `NodeGraphWidget` 改动
- 新增字段：`ConnectionDrag pending`（package-private，初始 null）。
- 新增嵌套：`static final class ConnectionDrag { final NodeLayout fromLayout; final int outIdx; final double fromWx; final double fromWy; final int color; }` —— 不存鼠标坐标，鼠标坐标从 renderWidget 的 mouseX/mouseY 取（实时）。outIdx>=0 表示从输出端口拖（输出→输入，合法方向）；若需输入→输出拖拽（TaskH），TaskH 在设置时自行决定 fromWx/fromWy 的含义。TaskG 只画 from→mouse 的曲线。
  - 简化：`ConnectionDrag` 只存 (fromLayout, outIdx, color)。from 锚点 = fromLayout.outputPort(outIdx)。to = 鼠标世界坐标。
- `renderWidget` 改：renderGrid → **renderConnections(g)** → renderNodes（顺序，D1）→ disableScissor → 预览线（在 scissor 外不画，预览也应在画布内，故放 enableScissor 内、disableScissor 前，紧跟 renderNodes 之后）→ tooltip。
  - 修正：预览线在 scissor 内渲染（画布裁剪），顺序：renderGrid → renderConnections → renderNodes → renderPending → disableScissor → tooltip。
- `protected void renderConnections(GuiGraphics g)`：遍历 graph.connections()，对每条：from=graph.node(c.fromNode())、to=graph.node(c.toNode())；new NodeLayout(from)/NodeLayout(to)；color=c.isAutoConverted()?WARN_COLOR:fromLayout.outputPortData(c.fromOutput()).value().type().color()；ConnectionRenderer.render(...)；if(c.isAutoConverted()) ConnectionRenderer.renderWarnMark(g, 中点屏幕坐标)。
- `protected void renderPending(GuiGraphics g, int mouseX, int mouseY)`：if pending==null return；from 锚点=pending.fromLayout().outputPort(pending.outIdx())；to 世界=viewport.screenToWorld(mouseX/Y)；color=applyAlpha(pending.color(), PREVIEW_ALPHA)；renderPreview(...)。

## 不变量
1. 连线渲染不改 NodeGraph 数据（纯读）。
2. 端口索引越界（fromOutput/outIdx 超节点 outputs）→ IndexOutOfBoundsException 透传（NodeLayout.outputPort 抛）。
3. 连线两端节点缺失 → NodeGraph.node 抛 IllegalArgumentException 透传（数据层保证一致性，不应发生）。
4. pending 状态只在内存，不持久化、不影响 NodeGraph。
5. 屏幕坐标采样步长恒定 STEP，与 scale 无关（粗细恒定）。

## 异常
- 上述透传异常（IndexOutOfBounds/IllegalArgument）不在渲染层吞，向上抛（渲染期不应触发；触发即数据层 bug）。

## 副作用
- 仅绘制，无数据变更。
- pending 由 widget 内存持有，TaskH 通过 package-private setter（或直接字段，同包 client.widget）设置。

## 决策记录
1. **不用 Tesselator/LINE_STRIP**：1.20.1 LINES 模式需 POSITION_COLOR_NORMAL + RenderType.lines() shader + lineWidth 驱动限制；本环境无 runClient 验证条件，风险高。fill 路径已被 TaskF 验证，可靠。
2. **段包围盒 fill 而非点画**：点画在步长处有缝隙/锯齿；段包围盒（相邻点轴对齐矩形）重叠填充，无缝连续，对水平流向曲线视觉良好。
3. **屏幕坐标而非世界坐标**：世界坐标下 fill 会被 pose.scale 缩放导致粗细随 scale 变化（缩小断裂、放大过粗）；屏幕坐标粗细恒定。
4. **自动转换橙色覆盖类型色**：类型色可能与警告语义冲突；橙色 + 中点方块标记双保险。
5. **预览线半透明而非虚线**：虚线需手动跳段，复杂；半透明（alpha=0x80）实现简单且视觉区分度足够。
6. **renderConnections 独立按需建 NodeLayout**（不与 renderNodes 共享缓存）：连线两端节点通常都在可见区；重复建 NodeLayout（无 Font 依赖，纯几何）开销可忽略；保持对 TaskF renderNodes 的最小改动。
7. **pending 数据源归 TaskH**：TaskG 只交付渲染能力 + widget 字段；拖拽起止逻辑归 TaskH，避免 TaskG 越界。
8. **采样段数自适应**：N=clamp(round(屏幕距离/STEP),12,64)，短线不冗余、长线不断裂。

## 验证
- 客户端类 compileJava 通过（无 runClient 实测条件，视觉由用户 runClient 确认）。
- 无 JUnit（渲染层无法单测视觉效果；NodeLayout 几何已在 TaskF 验证）。
- 结构正确性：renderWidget 顺序（D1）、配色规则（D3）、pending 渲染条件（pending!=null）。
- 全工程测试数不变（100），0 失败 0 错误。
