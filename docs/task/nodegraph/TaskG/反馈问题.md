# TaskG 反馈问题

## 1. MIN_CURVE_DX 单位：规格"世界单位" vs 实现"屏幕像素"
- 规格决策记录 D5：`MIN_CURVE_DX=24.0`（世界单位）。
- 实现（ConnectionRenderer.controlDx）：控制点在**屏幕坐标空间**定义（与 D2"屏幕坐标绘制、粗细恒定"一致），MIN_CURVE_DX 当**屏幕像素**用。
- 原因：控制点水平偏移若按世界单位，经 scale 缩放后屏幕偏移会随缩放变化（缩小→偏移过小→曲线过平），与"粗细恒定、视觉一致"的屏幕坐标策略不统一。
- 影响：视觉行为符合预期（近距离节点始终有足够水平偏移）；语义为屏幕像素而非世界单位。
- 以实现为准（视觉一致性优先）。

## 2. renderPreview y 分量笔误（已当场修正，非遗留）
- 首版 renderPreview 的 qy 误写为 `3*u*u*t*sy0 + 3*u*t*t*sy3`（省略 c1y/c2y 变量直接代值，数值上与 c1y=sy0/c2y=sy3 等价，但与 render 不一致且可读性差）。
- 修正为统一用 c1y/c2y 变量，与 render 公式一致。无行为变化。

## 3. MC 1.20.1 画线 API 缺失（实现备忘，非规格错误）
- GuiGraphics 无任意角度画线原语（hLine/vLine 本质 fill）。规格 D2 决策记录已说明不采用 Tesselator/LINE_STRIP（需 POSITION_COLOR_NORMAL + shader + lineWidth 驱动限制，无 runClient 验证条件）。
- 采用"屏幕坐标贝塞尔采样 + 段包围盒 fill"，复用已验证 fill 路径。锯齿由小步长（STEP=2px）与相邻段包围盒重叠缓解，对水平流向曲线视觉良好。
