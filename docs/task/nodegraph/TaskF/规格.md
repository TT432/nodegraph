# TaskF — 节点渲染（客户端）

## 目标

在 TaskE 画布上渲染 `NodeGraph` 中的节点：节点头、节点体、体部输入组件（InputWidget）、
输入/输出端口（Port）。按端口 `Type.color()` 自动配色；端口旁标记名字；悬浮端口/输入组件时
显示 tooltip（类型 + 描述）。本任务只做**渲染 + 命中检测几何**，不做交互（拖动/连线归 TaskH）。

## 前置

- TaskE：`NodeGraphWidget`（renderWidget 钩子）、`Viewport`（world↔screen 变换）。
- TaskA：`Node`（header/x/y/widgets/inputs/outputs）、`Port`（key/value/name/type/description）、
  `InputWidget`（key/value/kind/name/type/description/currentValue）、`TypedValue`、`Type.color()`。
- MC 1.20.1 客户端 API（已核实 sources jar）：
  - `GuiGraphics.drawString(Font, Component, int x, int y, int color)` / `(…, boolean dropShadow)`。
  - `GuiGraphics.drawCenteredString(Font, Component, int x, int y, int color)`。
  - `GuiGraphics.fill(int,int,int,int,int)`、`renderOutline(int x,int y,int w,int h,int color)`。
  - `GuiGraphics.renderTooltip(Font, Component, int mouseX, int mouseY)`（单行）。
  - `GuiGraphics.renderComponentTooltip(Font, List<Component>, int mouseX, int mouseY)`（多行）。
  - `GuiGraphics.pose()->PoseStack`；`PoseStack.translate/scale/pushPose/popPose`。
  - `Font`：`public final int lineHeight = 9;`、`int width(Component/String/FormattedCharSequence)`。
  - `Minecraft.getInstance().font`（public final Font）。
  - `Component.literal(String)`、`Component.empty()`。

## 关键设计决策

### D1. 节点几何 = 世界坐标（随缩放缩放）
节点尺寸用"世界单位"表达（= scale=1 时的像素）。渲染时 `pose.translate(screenX,screenY)` +
`pose.scale(s,s,1)`，之后用世界单位绘制；文字经 PoseStack 缩放自然随之缩放。与 TaskE 视口
模型一致（Node.x/y 已是世界坐标）。

### D2. 固定节点宽度；高度按内容增长
`NODE_WIDTH = 120` 常量。高度 = HEADER_HEIGHT + widgets×ROW_HEIGHT + max(inputs,outputs)×ROW_HEIGHT。
固定宽度使端口锚点 x 恒为节点左右边，简化连线几何（TaskG）与命中检测。文本超长不换行/截断
（接受溢出；后续可加省略号，非本任务范围）。

### D3. NodeLayout 为纯几何类（无 Font/MC 渲染依赖）
仅依赖 `api.model.Node`（+ Component，公共侧）。提供所有矩形/锚点的世界坐标 getter。
可 JUnit 验证坐标数学。NodeRenderer、TaskG（连线锚点）、TaskH（命中检测）共用同一几何源，
保证一致。

### D4. 端口以"填充小方块"近似圆点
MC `GuiGraphics` 无圆形原语；用边长 `2*PORT_RADIUS` 的轴对齐填充方块表示端口。视觉上读作
"端口点"，类型由 `Type.color()` 区分。圆形端口可用 RenderSystem+buffer 自绘，留待后续打磨。

### D5. 体部布局：组件在上，端口行在下
- 头部条：节点最顶部，full width。
- 体部：头部下方。先纵向堆叠所有 `InputWidget`（每个一行：名字左 + 当前值文本右），
  再是端口区。端口区每行：左=输入端口点+标签，右=输出端口点+标签；行数 = max(inputs,outputs)。
- 输入端口锚点在节点**左边界**（x=节点左），输出端口锚点在**右边界**（x=节点右）。
  符合用户"输入在左侧，输出在右侧"。

### D6. 头部配色：固定（不按类型）
用户"按类型自动配色"针对端口/连线，不针对节点头。头部用固定色（比体部稍亮），文字用白色。
节点间区分依赖头的 Component 文本与端口颜色。后续可按 definition id 哈希着色，非本任务范围。

### D7. InputWidget 仅静态展示（不做交互编辑）
每个 widget 渲染一行：`name : currentValue.toString()`。`InputWidgetKind`（TEXT/SLIDER/
BUTTON_GROUP/DROPDOWN）的差异化可视化（如 SLIDER 画进度条、DROPDOWN 画下拉箭头）属交互
渲染，归 TaskH；本任务统一文本展示，规格记录此边界。

### D8. Tooltip 在 scissor 之外渲染
节点体在 `enableScissor` 内绘制（裁剪到画布），但 tooltip 不应被画布裁剪。故：渲染节点时
记录"待显示 tooltip"，在 `disableScissor()` 之后用屏幕坐标 `renderComponentTooltip` 绘制。

### D9. Tooltip 内容 = 名字 + 类型 + 描述
悬浮端口或输入组件时，tooltip 三行：`[name]`、`Type: <type.id()>`、`[description]`。用
`renderComponentTooltip(font, List<Component>, mouseX, mouseY)`。鼠标同时只命中一个目标，
取最上层（端口优先于 widget 行）。

### D10. 可见性剔除
节点屏幕包围盒完全在画布矩形外则跳过渲染（少数节点非必要，但廉价且为 TaskK 演示密度兜底）。

## 交付物

### 1. `com.nodegraph.nodegraph.client.layout.NodeLayout`（final，纯几何）
```
常量：
  NODE_WIDTH      = 120.0
  HEADER_HEIGHT   = 14.0
  ROW_HEIGHT      = 14.0
  PORT_RADIUS     = 5.0   // 端口方块半边长（方块边长 2*R）
  PORT_HIT_RADIUS = 9.0   // 命中半径（世界单位），略大于视觉半径便于点中
  PADDING         = 4.0
  HEADER_COLOR    = 0xFF3A3A3A
  BODY_COLOR      = 0xFF2A2A2A
  OUTLINE_COLOR   = 0xFF6A6A6A
  OUTLINE_HOVER   = 0xFFE0E0E0
  TEXT_COLOR      = 0xFFFFFFFF
  WIDGET_VALUE_COLOR = 0xFFBBBBBB

嵌套 record：
  Rect(double x,double y,double w,double h){ boolean contains(double,double); }
  PortAnchor(double x,double y,int index)

构造：NodeLayout(Node node)
方法：
  Node node()
  double width()                // NODE_WIDTH
  double height()               // HEADER_HEIGHT + widgets*ROW + portRows*ROW
  Rect bounds()                 // (node.x, node.y, NODE_WIDTH, height)
  Rect header()                 // (x, y, NODE_WIDTH, HEADER_HEIGHT)
  Rect body()                   // (x, y+HEADER_HEIGHT, NODE_WIDTH, height-HEADER_HEIGHT)
  int widgetRowCount()
  Rect inputWidget(int i)       // (x, y+HEADER_HEIGHT+i*ROW, NODE_WIDTH, ROW)
  int portRowCount()            // max(inputs.size, outputs.size)
  double portRowsTop()          // 体部内端口区起始 y
  PortAnchor inputPort(int i)   // 锚点 (x, portRowsTop + i*ROW + ROW/2)
  PortAnchor outputPort(int i)  // 锚点 (x+NODE_WIDTH, portRowsTop + i*ROW + ROW/2)
  Rect inputPortLabel(int i)    // 标签区（点右侧到中部）
  Rect outputPortLabel(int i)   // 标签区（中部到点左侧）
  // 命中检测（世界坐标）：
  Optional<Integer> pickInputPort(double wx,double wy)
  Optional<Integer> pickOutputPort(double wx,double wy)
  Optional<Integer> pickInputWidget(double wx,double wy)
  boolean headerContains(double wx,double wy)
```
不变量：所有矩形/锚点为世界坐标；`contains` 闭区间含边界；`pick*` 用 PORT_HIT_RADIUS 圆形
判定（距离 ≤ 半径）；widget/标签用矩形 `contains`。

### 2. `com.nodegraph.nodegraph.client.render.NodeRenderer`（final）
职责：用 `NodeLayout` + `GuiGraphics` + `Font` + `Viewport` 绘制单个节点。
```
构造：NodeRenderer()  // 无状态工具类，可 private 构造 + 静态方法
静态：
  void render(GuiGraphics g, Font font, NodeLayout layout, Viewport vp,
              int originX, int originY, boolean hovered)
  Optional<TooltipLines> pickHover(NodeLayout layout, Viewport vp,
              int originX, int originY, int mouseX, int mouseY)
```
`render` 步骤：
1. 计算节点左上屏幕坐标 `sx=vp.worldToScreenX(node.x,originX)`、`sy=vp.worldToScreenY(node.y,originY)`。
2. `g.pose().pushPose()`；`translate(sx,sy,0)`；`scale(s,s,1)`（s=vp.scale()）。
3. 在世界单位（原点=节点左上）下：
   - `fill(0,0,NODE_WIDTH,height, BODY_COLOR)` 体背景。
   - `fill(0,0,NODE_WIDTH,HEADER_HEIGHT, HEADER_COLOR)` 头部。
   - `drawString(font, node.header, PADDING, (HEADER_HEIGHT-9)/2, TEXT_COLOR)` 头文字（左对齐）。
   - `renderOutline(0,0,NODE_WIDTH,height, hovered?OUTLINE_HOVER:OUTLINE_COLOR)` 描边。
   - 每个 InputWidget：`drawString(font, name, PADDING, widgetY+pad, TEXT_COLOR)`；
     `drawString(font, String.valueOf(currentValue), NODE_WIDTH-PADDING-font.width(valStr), …, WIDGET_VALUE_COLOR)` 右对齐。
   - 每个输入端口：`fill(anchor.x-R, anchor.y-R, anchor.x+R, anchor.y+R, type.color())`；
     标签 `drawString(font, port.name, anchor.x+R+2, anchor.y-(9/2), TEXT_COLOR)`。
   - 每个输出端口：方块同理在右锚点；标签 `drawString(font, port.name, anchor.x-R-2-font.width(name), …)` 右对齐。
4. `g.pose().popPose()`。

`pickHover`：将 mouseX/mouseY 转世界坐标，依次试 `pickInputPort`→`pickOutputPort`→`pickInputWidget`
（端口优先），命中则返回该 TypedValue 的 (name,type,description) 供 tooltip。无命中→Optional.empty。

`TooltipLines`：包级私有 record(List<Component> lines)，由调用方在 scissor 外 renderComponentTooltip。

注：`font.width` 接受 Component/String；右对齐用 String 值。drawString 在缩放的 PoseStack 下自动缩放。

### 3. `NodeGraphWidget` 改动
- 新字段：`private Font font`（构造时 `Minecraft.getInstance().font` 缓存，避免每帧查）。
  构造签名不变（仍 (x,y,w,h,graph)）。
- `renderWidget`：在 `renderGrid(g)` 之后、`disableScissor()` 之前调 `renderNodes(g, mouseX, mouseY)`；
  之后（scissor 外）渲染待显示 tooltip。
- `renderNodes`：遍历 `graph.nodes()`，对每个 node 构造 `NodeLayout`，做可见性剔除
  （屏幕包围盒与画布矩形不相交则跳过），判定 hovered（鼠标是否在节点 bounds 内，用于描边高亮），
  调 `NodeRenderer.render`，并调 `NodeRenderer.pickHover` 收集 tooltip（首个命中即停）。
- 节点鼠标 hovered 判定用世界坐标 `pickHover` 同源转换。
- 新 public 方法 `viewport()` 已存在；不新增其他 public API。

### 4. 不修改 NodeGraph / api 层
NodeGraph 只读引用。

## 不变量

1. NodeLayout 所有几何量在世界坐标系，仅依赖 Node 内容 + 常量，纯函数式（每次构造独立）。
2. 输入端口锚点 x == node.x（左边界）；输出端口锚点 x == node.x + NODE_WIDTH（右边界）。
3. 端口行 i（0-based）的中心 y == portRowsTop + i*ROW_HEIGHT + ROW_HEIGHT/2，对输入输出一致。
4. NodeLayout.height() == HEADER_HEIGHT + widgetCount*ROW_HEIGHT + portRowCount*ROW_HEIGHT。
5. pick* 命中判定：端口圆形（距离 ≤ PORT_HIT_RADIUS）；widget/header 矩形 contains。
6. 节点渲染包裹在 pushPose/popPose 内，缩放后状态完全还原（不泄漏变换到下一节点或网格）。
7. Tooltip 总在 disableScissor 之后渲染，不被画布裁剪。
8. 同一帧最多渲染一个 tooltip（多目标取首命中）。

## 异常

- NodeLayout 构造：`Objects.requireNonNull(node)`。
- inputPort/outputPort/inputWidget 索引越界 → `IndexOutOfBoundsException`（由底层 List 访问抛出）。
- NodeRenderer.render：font/g/layout/vp 非 null 校验（NullPointerException via requireNonNull）。

## 副作用

- NodeLayout：纯计算，无副作用。
- NodeRenderer.render：仅通过 GuiGraphics 绘制（含 PoseStack push/pop 平衡）。
- NodeGraphWidget：新增每帧遍历节点绘制；不改 NodeGraph 数据。

## 决策记录

1. **几何独立于 Font**：固定宽度策略使 NodeLayout 不需 Font 即可定出端口锚点 → 可 JUnit、
   可被 TaskG/H 复用。文本宽度仅用于右对齐视觉，不影响几何正确性。
2. **方块端口**：MC 无圆原语；方块是诚实最小实现，圆形为后续打磨项（已记录，非退化）。
3. **头部不按类型配色**：规格明确"按类型配色"对象是端口/连线；节点头用固定色。
4. **InputWidget 统一文本展示**：差异化可视化与交互耦合，归 TaskH；本任务不假装支持半成品 UI。
5. **Tooltip scissor 外渲染**：避免被画布裁掉，是必要的视觉正确性。
6. **portRows = max(inputs,outputs)**：两侧端口共享行空间，行中心 y 对齐，简化连线锚点。
7. **可见性剔除**：O(nodes) 廉价；演示期节点少但为后续密集图兜底。
8. **hovered 高亮描边**：仅视觉提示；选择集高亮（TaskI）将复用 OUTLINE_HOVER 常量。

## 验证清单（JUnit 5 + compileJava）

### TestNodeLayout（纯几何，~12 测试）
1. `emptyNodeHeightIsHeaderOnly`：0 widget + 0 port → height==HEADER_HEIGHT。
2. `heightGrowsWithWidgets`：k widget + 0 port → height==HEADER+k*ROW。
3. `heightGrowsWithMaxPorts`：inputs=2,outputs=5 → portRows=5，height 含 5*ROW。
4. `headerRectAtNodeOrigin`：header==(x,y,NODE_WIDTH,HEADER_HEIGHT)。
5. `bodyRectBelowHeader`：body.y==y+HEADER_HEIGHT；body.h==height-HEADER_HEIGHT。
6. `inputPortAnchorOnLeftEdge`：inputPort(i).x==node.x。
7. `outputPortAnchorOnRightEdge`：outputPort(i).x==node.x+NODE_WIDTH。
8. `portAnchorYCenteredInRow`：行 i 中心 y==portRowsTop+i*ROW+ROW/2，输入输出同 y。
9. `pickInputPortHitsWithinRadius`：锚点处及偏移<半径命中；>半径不命中。
10. `pickOutputPortHitsWithinRadius`：同上，输出侧。
11. `pickInputWidgetByRow`：widget 行矩形内命中对应索引；行外不命中。
12. `headerContains`：header 矩形内 true、外 false。
