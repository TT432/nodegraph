# TaskJ 节点组（客户端）— 规格

## 目标
为 `NodeGroup` 交付客户端渲染与交互（数据模型 TaskA、创建命令 TaskI 已就绪）：
- 线框包围多节点 + 头部条（`Component`）。
- 拖组头 → 移动组框及内部全部成员节点。
- 拖节点进入组范围 → 放入（成员归属变更）；拖出范围 → 移出。
- 可放大缩小：拖右下角 resize handle 调整包围盒 `width/height`；Ctrl+滚轮 hover 组头调整 `scale`。
- 组可被选中（参与 SelectionModel.groups），框选/Shift+组头/点击组头。

## 前置
- **TaskA**：`NodeGroup`(id/header/x/y/width/height/scale/setHeader/setPosition/setSize/setScale/contains(px,py))、`NodeGraph`(groups()/members(gid)/setNodeGroup(nodeId,gidOrNull)/group(gid))、`Node.groupId()`。
- **TaskC**：`SetGroupTransformCommand(graph,target,newX,newY,newW,newH,newScale)`、`SetNodeGroupCommand(graph,node,newGroup)`、`SetGroupHeaderCommand`、`MoveNodeCommand(graph,target,oldX,oldY,newX,newY)` 全量构造、`CompositeCommand(description,List<Command>)`、`UndoManager`。
- **TaskF**：`NodeLayout.bounds()->Rect(x,y,w,h)`、`NodeLayout.NODE_WIDTH/height()`。
- **TaskH**：`NodeInteractionController`（State IDLE/DRAG_NODE/DRAG_CONNECTION；onMouseClicked/Dragged/Released(mx,my,button)->boolean；button!=0 返回 false）。
- **TaskI**：`SelectionModel`(groups()/containsGroup/addGroup/toggleGroup/clear)、`SelectionController`(isSelecting/start/drag/finish(additive)/minWX/minWY/maxWX/maxWY)、`NodeGraphWidget`(selection()/graph()/viewport()/undo()/worldX/worldY/selectSingle)。
- MC 1.20.1：`GuiGraphics.fill/renderOutline/drawString`、`Screen.hasControlDown/hasShiftDown`、`Viewport.worldToScreenX/Y/screenToWorldX/Y`。

## 关键设计决策
**D1 渲染层次**：组作为分组背景，画在 grid 之上、connections 之下：
`bg → grid → renderGroups → connections → nodes → selectionBox → pending → disableScissor → tooltip → menu`。
节点与连线画在组线框之上；组头是线框顶部条，成员节点通常在头部下方，不遮挡。

**D2 交互优先级（pick 顺序，与渲染层独立）**：
mouseClicked(button0) 依次：端口(pickOutput/OutputPort) → 节点头(pickHeader) → 组 resize handle(pickResizeHandle) → 组头(pickGroupHeader)。首个命中接管；均未命中 → 空白框选。
即：节点在组之上，点节点头→拖节点；点组头露出部分→拖组。

**D3 组头拖动**：命中组头（无 Shift）→ DRAG_GROUP，记录组 id、按下世界点、组原 x/y、各成员原 x/y。拖动：组新位=组原+(curW-pressW)；各成员新位=成员原+(curW-pressW)，实时 setPosition（组+成员）。释放：位移≠0 → `CompositeCommand("Move group", SetGroupTransformCommand(graph,gid,newX,newY,oldW,oldH,oldScale) ∪ {MoveNodeCommand(graph,member,oldMX,oldMY,newMX,newMY) for 成员})`；位移=0 → selectSingleGroup(gid)。Shift+组头 → toggleGroup 不拖动。

**D4 resize handle**：组框右下角 `[x+w-HANDLE, y+h-HANDLE, +HANDLE, +HANDLE]`（HANDLE=8 世界单位）小方块。命中（无 Shift）→ DRAG_RESIZE_GROUP，记录组 id、原 x/y/w/h、按下世界点。拖动：newW=max(MIN_GROUP_W, oldW+(curW-pressW))；newH=max(MIN_GROUP_H, oldH+(curW...))；实时 setSize。释放：尺寸变化 → `SetGroupTransformCommand(graph,gid,oldX,oldY,newW,newH,oldScale)`。MIN_GROUP_W/H=40/含组头+若干行。

**D5 scale 调整**：Ctrl+滚轮 且鼠标在世界坐标组头区域内 → 调整该组 scale（delta>0 增大），clamp [MIN_GROUP_SCALE=0.5, MAX_GROUP_SCALE=2.0]，实时 setScale；释放不适用（滚轮即时）→ 每次 wheel 产生一条 `SetGroupTransformCommand(graph,gid,curX,curY,curW,curH,newScale)`。Ctrl+滚轮不在组头 → 画布缩放（TaskE 原逻辑）。scale 渲染语义=组框描边线宽（`lineWidth=clamp(round(BASE_LINE*scale),1,4)`）+ 组头标题右侧显示 `(scale%)`；**不影响成员节点坐标**（成员坐标缩放留作后续打磨，避免破坏 NodeLayout 世界坐标一致性）。

**D6 拖节点入组/出组（释放判定）**：DRAG_NODE 释放时，除 MoveNodeCommand 外，据节点最终中心点 `(endX+NODE_WIDTH/2, endY+layout.height()/2)` 判定组归属：遍历 `graph.groups()`，取**迭代序第一个** `contains(中心)` 的组作为 newGroup（D10），无则 null。若 newGroup != 节点当前 groupId → 追加 `SetNodeGroupCommand(graph,node,newGroup)`。位移≠0 且归属变 → `CompositeCommand("Move node", MoveNode, SetNodeGroup)`；位移≠0 归属不变 → MoveNodeCommand 单条；位移=0 → selectSingle（TaskH 原逻辑，不做归属判定）。
注：拖组头移动组+成员时成员随组平移、相对位置不变，不触发入组/出组。

**D7 GroupPick 纯逻辑工具类**（`client.interaction.GroupPick`，仅依赖 api.model，可 JUnit）：`findGroupContaining(NodeGraph,double wx,double wy)->Optional<NodeGroupId>`（点 vs 各组 contains，迭代序首个）、`findGroupHeader(NodeGraph,double wx,double wy)->Optional<NodeGroupId>`（点 vs 组头条区域）、`findResizeHandle(NodeGraph,double wx,double wy)->Optional<NodeGroupId>`（点 vs 右下角 handle）。组头条高度 `GROUP_HEADER=14`、handle `RESIZE_HANDLE=8` 作 GroupPick public 常量（与 NodeGroupRenderer 共用）。controller 与 widget 均通过 GroupPick 调用，单一几何源。

**D8 框选扩展选组**：`SelectionController.finish(additive)` 在原有节点相交判定的基础上，增加组相交判定：遍历 `graph.groups()`，组矩形 `(gx,gy,gw,gh)` 与框选世界矩形相交 → 命中组集合。additive → `selection.addGroup(each)`；否则 `selection.setGroups(hits)`（与 setNodes 同时替换）。组与节点各自独立 set/add。

**D9 组选择操作**：widget 加 `selectSingleGroup(NodeGroupId)`（clear + addGroup）。Shift+组头 → `selection.toggleGroup(gid)`（controller 内）。组选中高亮 = renderGroups 内据 `selection.containsGroup(gid)` 用 `SELECTED_COLOR` 描边。删除组（TaskI delete 已含 RemoveGroupCommand×selection.groups）对选中组生效。

**D10 多组重叠归属**：节点中心点可能落在多个重叠组内。取 `graph.groups()` 迭代序第一个 contains 的组（不取"最小"或"最内"，保持确定性与简单性）。记录：用户若需嵌套组语义另议。

## 交付物
1. **`client.render.NodeGroupRenderer`**（final，private 构造，无状态工具类）：
   - 常量 `GROUP_HEADER=14.0`、`RESIZE_HANDLE=8.0`、`BASE_LINE=1.5`、`COLOR_GROUP_FILL=0x284060A0`、`COLOR_GROUP_OUTLINE=0xFF6080C0`、`COLOR_GROUP_HEADER=0xFF3A5AA0`、`COLOR_GROUP_TEXT=0xFFFFFFFF`、`COLOR_RESIZE_HANDLE=0xFF6080C0`、`MIN_GROUP_SCALE=0.5`、`MAX_GROUP_SCALE=2.0`。
   - `static render(GuiGraphics g, Font font, NodeGroup group, Viewport vp, int originX, int originY, boolean selected)`：pushPose+translate(screen x,y)+scale(vp.scale)；fill 半透明框 + renderOutline 描边（selected?SELECTED_COLOR:COLOR_GROUP_OUTLINE，线宽据 scale 系数）+ 头部条 fill+drawString(header) + 标题右侧 scale% 标签（scale≠1.0 时）+ 右下角 resize handle 方块；popPose。
   - 与 NodeRenderer 同样的世界坐标+viewport scale 渲染模型。
2. **`client.interaction.GroupPick`**（final，private 构造，纯逻辑工具类，仅依赖 api.model）：
   - 常量 `GROUP_HEADER=14.0`、`RESIZE_HANDLE=8.0`（与 NodeGroupRenderer 一致）。
   - `static Optional<NodeGroupId> findGroupContaining(NodeGraph,double wx,double wy)`。
   - `static Optional<NodeGroupId> findGroupHeader(NodeGraph,double wx,double wy)`（组头区域=组框顶部 GROUP_HEADER 高度）。
   - `static Optional<NodeGroupId> findResizeHandle(NodeGraph,double wx,double wy)`（右下角 RESIZE_HANDLE 方块）。
3. **`NodeInteractionController` 改**：State 加 `DRAG_GROUP, DRAG_RESIZE_GROUP`。
   - onMouseClicked(button0)：端口→节点头→`GroupPick.findResizeHandle`→`GroupPick.findGroupHeader`。组头命中：Shift→`widget.selection().toggleGroup(gid)` 不拖动；否则 startDragGroup。resize handle 命中→startDragResize。
   - onMouseDragged：DRAG_GROUP→移动组 setPosition + 成员 setPosition；DRAG_RESIZE_GROUP→组 setSize（clamp MIN）。
   - onMouseReleased：DRAG_GROUP→位移≠0 CompositeCommand(SetGroupTransform+MoveNode×成员)，位移=0 selectSingleGroup；DRAG_RESIZE_GROUP→尺寸变 SetGroupTransformCommand；DRAG_NODE→入组/出组判定（D6）。
4. **`NodeGraphWidget` 改**：
   - renderWidget 加 `renderGroups(g)`（在 renderGrid 之后、renderConnections 之前）：遍历 graph.groups()，selected=selection.containsGroup(gid)，NodeGroupRenderer.render。
   - mouseScrolled：Ctrl 且 `GroupPick.findGroupHeader(graph,worldX,worldY)` 命中 → 调整该组 scale（clamp，SetGroupTransformCommand）；否则原画布缩放。
   - 加 `selectSingleGroup(NodeGroupId)`（clear+addGroup）。
5. **`SelectionController` 改**：finish 扩展，组相交判定 → selection.addGroup/setGroups（D8）。
6. NodeGroup 不变；NodeGraph 不变。

## 不变量
1. 组渲染在 connections 之下、grid 之上（背景层）。
2. 拖组头时成员相对组框的位置保持不变（组与成员同偏移平移）。
3. 入组/出组判定只在 DRAG_NODE（单独拖节点）释放时发生；拖组头/resize 不触发成员归属变更。
4. resize 后 width/height ≥ MIN_GROUP_W/H（40/34）。
5. scale ∈ [0.5,2.0]，clamp。
6. GroupPick 三个方法均不依赖 MC 渲染类（纯 api.model），可 JUnit。
7. 多组重叠时 findGroupContaining 取迭代序首个。

## 异常
- 未知 NodeGroupId：GroupPick 内 `graph.group(gid)` 不调用（只用 group.x()/y()/width()/height()，遍历 groups() 已知存在）；controller 移动成员用 `graph.node(memberId)`（TaskH 已验证存在）。
- resize 到极小：clamp 至 MIN，不抛异常。
- scale 越界：clamp，不抛异常。

## 副作用
- 组头拖动释放：1 条 CompositeCommand 入 undo 栈。
- resize 释放：1 条 SetGroupTransformCommand 入 undo 栈。
- scale 滚轮：每次 wheel 1 条 SetGroupTransformCommand（即时生效；undo 逐档回退）。
- 节点入组/出组：并入移动 CompositeCommand 或独立 SetNodeGroupCommand（位移=0 但归属变？不会——位移=0 走 selectSingle 不判定归属）。

## 决策记录
- scale 不绑定成员坐标缩放（D5）：避免与 NodeLayout 世界坐标几何冲突；保留字段与命令供未来扩展，当前作线宽+标签视觉系数。
- resize handle 而非边框任意拖拽：右下角单一 handle 简化命中与状态机；四角/边拖拽留作打磨。
- 入组判定用节点中心点而非整个 bounds：中心点简单且符合"节点主要部分在组内"直觉；bounds 完全包含判定过严（节点部分压线即不入组反不直观）。
- 框选选组用矩形相交（非完全包含）：与节点框选一致（TaskI SelectionController 用相交）。
- GroupPick 抽为纯逻辑类（D7）：与 NodeLayout 同理，几何判定可测、单一源、controller/widget 共用。
- scale 滚轮命令逐档：简单；合并连续滚轮为单命令需时间窗聚合，复杂度高，留作打磨。
