# TaskI — 选择与右键菜单（客户端）

## 目标
在 `NodeGraphWidget` 上实现：
1. **左键拖空白=框选**多节点（替换 TaskE 的"左键空白=pan"，pan 改由中键独占）。
2. **右键菜单**：动态菜单项（复制/剪切/粘贴/删除/组合），点击执行并关闭。
3. **键盘快捷键**：Ctrl+C/X/V、Del、Ctrl+Z、Ctrl+Shift+Z，接入 `UndoManager`。
4. **选中可视化**：选中节点描边高亮 + 框选矩形（半透明填充+边框）。
5. 每个操作可经 `UndoManager` 撤销/重做。

## 前置
- TaskA：NodeGraph/Node/NodeGroupId/NodeId。
- TaskC：UndoManager / CompositeCommand / RemoveNodeCommand / RemoveGroupCommand / CreateGroupCommand / SetNodeGroupCommand / MoveNodeCommand（全量构造）/ ConnectCommand / Command。
- TaskD：Clipboard（copy/cut/paste）。
- TaskE：NodeGraphWidget 事件钩子 / Viewport / NodeGraphScreen。
- TaskF：NodeLayout.bounds() / headerContains() / pick*。
- TaskH：NodeInteractionController（左键节点头拖动/端口连线）。

## 关键设计决策
### D1 — 事件路由重构（pan 归中键，左键归框选）
用户原始需求："左键拖→框选多节点；中键拖→拖动画布本身；右键→菜单"。当前 TaskE 实现为"左键空白=pan"与之冲突。TaskI 把左键空白改为框选，pan 仅中键(2)。右键(1)=打开菜单。`isValidClickButton` 调整为 {0,1,2}，但 button==1/2 不互相兼任。

**mouseClicked 新分支顺序**（边界内）：
1. `button==1`（右键）→ 打开 ContextMenu（先据命中决定单选该节点），return true。
2. `button==0 && interaction.onMouseClicked(...)` 命中节点头/端口 → return true（TaskH 不变）。
3. `button==0` 未命中 → `selection.startBoxSelect(world)`，return true（框选）。
4. `button==2`（中键）→ PANNING，return true。

### D2 — SelectionModel 纯 id 集合（纯 Java，可 JUnit）
`SelectionModel` 仅持有 `Set<NodeId>` + `Set<NodeGroupId>`（LinkedHashSet 保序），无 graph 依赖。框选相交逻辑（`pickInRect`：遍历 graph.nodes()，bounds() 与世界矩形相交）放客户端层（SelectionController），因为它依赖 NodeLayout（client 包）。SelectionModel 只做集合运算：add/remove/toggle/setAll/clear/contains/isEmpty/nodes()/groups()/size。

### D3 — SelectionController（框选状态机，客户端）
状态 `IDLE/BOX_SELECT`。`startBoxSelect(worldX,worldY)` 记录起点；拖动中 widget 调 `dragBox(curWorld)` 更新矩形；`finishBoxSelect(graph, additive)`：计算世界矩形（minX,minY,maxX,maxY）与各节点 bounds() 相交→`additive`(Shift)? 追加 : 替换 selection。矩形面积为 0（纯点击空白）→ 清空 selection。

### D4 — 点击节点头=单选（TaskH 协作）
左键点节点头由 `NodeInteractionController` 接管（拖动）。若释放时位移=0（纯点击），TaskH 调 `widget.selectSingle(nodeId)` → SelectionModel 清空+仅选该节点。框选空白释放且矩形面积=0 → 清空 selection。Shift 修饰：点击头=切换该节点选中态（不拖动若 Shift？为简化：Shift+点击头=toggle 选中且不进入拖动）。

> 决策：Shift+点击头=toggle 选中（不拖动）；无 Shift 点击头=若未拖动则单选，若拖动则移动。这通过 NodeInteractionController 在 onMouseClicked 检测 ShiftDown 实现。

### D5 — ContextMenu（自定义 overlay，无 MC 内置）
MC 1.20.1 无右键菜单组件。新建 `ContextMenu`（客户端 final class）：持有屏幕坐标 `(sx,sy)` + `List<MenuItem(label Component, enabled bool, Runnable)>`。`render(g,font)` 画背景矩形+每行文字（禁用项灰色）。`click(mouseX,mouseY)->boolean`：命中启用的行→执行 Runnable+返回 true（调用方负责关闭）；命中菜单矩形但禁用/无效→返回 false（保持打开）；命中菜单外→返回 false（调用方关闭菜单并 fallthrough）。

**菜单项规则**（右键时动态构造）：
- 复制（selection 非空）
- 剪切（selection 非空）
- 粘贴（`clipboard.hasContent()`）
- 删除（selection 非空）
- 组合（`selectedNodes.size()>1`）

空菜单（无以上任何可用项）→ 不打开。

**右键命中节点**：若该节点不在 selection → 清空 + 单选该节点；若在 selection → 保持（多选上下文菜单）。然后打开菜单。

### D6 — GroupNodesCommand（自洽，单步撤销）
TaskC 记录的限制：CreateGroupCommand 首次 execute 才分配 gid，无法与 SetNodeGroupCommand 同构 CompositeCommand。为让"组合"成为单步撤销单元，新建 `GroupNodesCommand`（api.command，纯 Java 可 JUnit），参考 PasteCommand 懒捕获模式：
- 构造 `(graph, Component header, double x, double y, double w, double h, List<NodeId> members)`。
- 首次 execute：`createGroup` 保存 group 对象；对每个 member `setNodeGroup(memberId, gid)`；executed=true。
- redo：`insertGroup(保存的 group)`；`setNodeGroup(memberId, gid)`×N。
- undo：`removeGroup(gid)`（级联清成员 groupId）；成员 groupId 被 removeGroup 置 null。
- description="Group"。

组的几何（x/y/w/h）：取选中节点 bounds 的包围盒（minX,minY,maxX,maxY-底），header 默认 `Component.literal("Group")`。位置在 widget.groupSelection() 里计算。

### D7 — 快捷键在 NodeGraphScreen.keyPressed
widget 是 GuiEventListener 但键盘焦点管理复杂；`NodeGraphScreen` 是键盘入口。override `keyPressed`：
- Ctrl+C → canvas.copy()
- Ctrl+X → canvas.cut()
- Ctrl+V → canvas.paste(at 视口中心或鼠标)
- Del(261) → canvas.delete()
- Ctrl+Z(90) 且 !Shift → undo.undo()
- Ctrl+Z 且 Shift → undo.redo()  （即 Ctrl+Shift+Z）
- ESC(256) → 若菜单打开则关闭菜单；否则 super（关闭屏）
- 其余 → super.keyPressed

键码用裸数字（67/86/88/90/261/256），与 MC 1.20.1 Screen 源码一致（已核实）。`hasControlDown/hasShiftDown` 用 `Screen.hasControlDown()`/`hasShiftDown()`。

### D8 — 操作方法集中在 widget（菜单+键盘共用）
`NodeGraphWidget` 暴露 public：
- `selection()` → SelectionModel
- `clipboard()` → Clipboard
- `copy()` / `cut()` / `paste()` / `delete()` / `groupSelection()`
- `selectSingle(NodeId)` / `clearSelection()`
- `openMenu(screenX, screenY)` / `closeMenu()` / `menu()` → ContextMenu 或 null
- `handleKey(int keyCode)` → boolean（供 screen.keyPressed 转发，内部调 copy/cut/paste/delete/undo/redo）

paste 偏移：以视口中心的世界坐标减去快照中心（近似），简化为固定小偏移 (16,16) 或视口中心。决策：paste at 视口左上角世界坐标 + (16,16) 偏移，避免与原节点重叠。

### D9 — 选中高亮 + 框选矩形渲染
`renderWidget` 顺序：背景→scissor→grid→connections→**nodes（选中节点叠加 SELECTED_OUTLINE）**→**框选矩形（BOX_SELECTING 时）**→pending→disableScissor→tooltip→**ContextMenu（在 scissor 外，顶层）**。
- 选中高亮：NodeRenderer.render 后，若 `selection.contains(node.id())`，用 `renderOutline` 叠加 SELECTED_COLOR=0xFFFFFF00（黄色）描边。
- 框选矩形：世界矩形→屏幕坐标，`fill` 半透明(0x40FFFFFF)+`renderOutline`(0xFFAAAAAA)。

### D10 — 删除=单 CompositeCommand（单步撤销）
`delete()`：若 selection 非空 → `undo.apply(new CompositeCommand("Delete", RemoveGroupCommand×groups + RemoveNodeCommand×nodes))`（复用 TaskD cut 的级联模式）。删除后清空 selection。

## 不变量
1. SelectionModel 中 NodeId 必须对应当前图中存在的节点（由调用方保证；model 不校验，避免与 graph 解耦失效）。框选/删除后，widget 主动调用 selection 与 graph 同步（删除后 clear）。
2. ContextMenu 打开期间，框选/拖动节点被抑制（menu != null 时事件优先关闭菜单）。
3. GroupNodesCommand.members 列表构造时固定，execute/redo 始终用同一 list。
4. GroupNodesCommand.undo 后 group 对象保留（供 redo insertGroup），不被 null 化。
5. 框选矩形面积为 0（minX==maxX 或 minY==maxY）→ 清空 selection（非追加，无论 Shift）。
6. isValidClickButton={0,1,2}；button==1 不进 pan/框选；button==2 不进框选/菜单。

## 异常
- GroupNodesCommand members 为 null → NullPointerException（构造 Objects.requireNonNull）。空 members 列表允许（仅建组无成员）。
- SelectionModel.add(null)/contains(null) → NullPointerException（值类型集合不接 null）。
- copy/cut/delete/groupSelection 在 selection 为空时为 no-op（不 apply 命令，不报错）。
- paste 在 clipboard 空时 no-op。

## 副作用
- copy/cut 调 Clipboard（覆盖单槽 snapshot）。
- paste/delete/group/cut 经 UndoManager.apply → 触发 listeners。
- 删除/剪切后 selection 被 clear（引用的节点已不在图中）。
- paste 后新节点 id 由 PasteCommand 分配，不自动选中（UI 可据返回值选；本 Task 不强求，留可选）。
- ContextMenu 执行项后自动关闭。

## 决策记录
1. 左键 pan → 中键 pan（D1）：尊重用户原始交互需求；TaskE 的左键 pan 是临时默认，TaskI 正式落地用户需求。
2. SelectionModel 纯 id（D2）：可 JUnit，与 graph 解耦，框选相交依赖 NodeLayout 故放 controller。
3. GroupNodesCommand 自洽（D6）：避免 TaskC 两步撤销限制，提供单步原子"组合"。
4. ContextMenu 自定义（D5）：MC 无内置，最小 overlay 实现足够。
5. 键盘在 Screen（D7）：widget 键盘焦点复杂，Screen 是天然键盘入口。
6. Shift+头=toggle（D4）：多选扩展；无 Shift=单选/拖动。
7. paste 偏移固定（D8）：简化，避免计算快照质心。
8. 删除=CompositeCommand（D10）：复用 TaskD cut 模式，级联连线由 RemoveNodeCommand 捕获。
