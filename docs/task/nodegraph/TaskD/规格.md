# TaskD 剪贴板与序列化

## 目标
为 NodeGraph 编辑器提供**剪贴板**（copy / cut / paste）与**选择快照序列化**（SelectionCodec）。
选择集合（一组节点 + 一组节点组）被编码为自描述、可重组的快照 DTO；粘贴时把快照重放为图中的新对象，**id 全部重映射**为新鲜 id，内部连线和组成员关系按原拓扑还原。paste / cut 均为**单一可撤销单元**（复用 TaskC 的 Command / UndoManager / CompositeCommand / RemoveNodeCommand / RemoveGroupCommand 与 TaskA 的低级恢复原语）。

纯 Java 层：仅依赖 TaskA 数据模型 + TaskC 命令系统 + `java.util` + `net.minecraft.network.chat.Component` / `net.minecraft.resources.ResourceLocation`（均为公共侧）。JUnit 可测。

- 复用的 TaskA API（精确签名）：

  - `Node`：`id()`、`definition()`、`header()`、`setHeader(Component)`、`x()`/`y()`、`widgets()`（List<InputWidget>）、`groupId()`。
  - `NodeGroup`：`id()`、`header()`、`x()`/`y()`/`width()`/`height()`/`scale()`、`setScale(double)`。
  - `InputWidget`：`key()`、`currentValue()`、`setCurrentValue(Object)`。

- 复用的 TaskC API：`Command`（execute/undo/redo/description）、`UndoManager`（apply/canUndo/canRedo 等）、`CompositeCommand`、`RemoveNodeCommand`、`RemoveGroupCommand`。

## 关键设计决策

### 1. 选择 = 显式 (nodes, groups) 二元组
- 选择集合由调用方（UI 层 TaskI）显式给定 `Collection<NodeId>` + `Collection<NodeGroupId>`。
- encode 只捕获选中节点、选中组、以及**两端节点均在选中集合内**的内部连线。
- 节点的 `groupId`：若该组**也在选中组集合**内，则记为本地组 id；否则记为 null（粘贴后该节点为自由节点，不引用未拷贝的组）。
- 组成员关系：派生自 `node.groupId`（单一真源）。粘贴后新组自动包含"其原组被选中且被粘贴"的节点子集——部分选中→粘贴组含子集，语义自洽。

### 2. 自描述快照：携带 definitions 映射，**不引入全局 NodeDefinitionRegistry**
- `SelectionSnapshot` 内含 `Map<ResourceLocation, NodeDefinition> definitions()`：encode 时从选中节点的 `definition()` 去重收集。
- 节点条目记 `defId`（ResourceLocation）；paste 从 snapshot 自带的 definitions 映射解析，**无需外部 registry 参数**。
- 理由：剪贴板是**进程内**对象；NodeDefinition 含 lambda（NodeFunction），无法磁盘序列化。TaskD 范围=内存剪贴板，快照自带 def 引用即完整自洽。磁盘持久化（需 registry 解析 defId→def）留待后续任务（TaskK 或独立持久化任务）。
- 这是经过权衡的决策：**不在 TaskD 引入 NodeDefinitionRegistry**，避免范围蔓延；快照自携带使 TaskD 自闭合。

### 3. id 重映射：本地 id → 新鲜全局 id
- encode 给每个选中节点/组分配**本地 id**（long，节点与组各自从 0 起的命名空间，互不冲突因分属不同 map）。连线与成员关系引用本地 id。
- paste（PasteCommand 首次 execute）调用 `graph.addNode` / `graph.createGroup` 分配**新鲜全局 id**，建立 `localId→newNodeId` / `localGid→newGroupId` 映射，据此重建连线与成员关系。
- 同一快照可粘贴多次：每次 addNode/createGroup 分配不同新鲜 id，互不冲突。

### 4. paste 为自洽 PasteCommand（实现 Command），**不预分配 id**
- 不引入 `NodeGraph.reserveNodeId` 等新 API。PasteCommand 复用 TaskA 已有的 `addNode`/`createGroup`（首次分配 id）+ `insertNode`/`insertGroup`/`addConnection`（redo 复用同一对象，id 稳定）。
- **懒捕获模式**（同 AddNodeCommand）：execute 内 `if (!executed)` 分支首次 addNode/createGroup + 应用覆盖（header/widget 值）+ setNodeGroup + addConnection 并保存对象；else 分支 redo：insertGroup → insertNode → addConnection，复用已保存对象。
- redo 路径顺序：先 insertGroup，再 insertNode（saved 节点对象的 groupId 字段在首次 execute 经 setNodeGroup 已固化，对象身份不变），最后 addConnection。成员关系随节点对象自动恢复，无需 redo 阶段再调 setNodeGroup。
- **不使用 CompositeCommand 组合 AddNodeCommand**：因为 AddNodeCommand 在 execute 时才分配 id，无法在构造期预解析出 ConnectCommand 所需的目标 id。PasteCommand 自洽地一次性完成"分配→建图→连线"，避开 id 预解析难题。
- PasteCommand 暴露 `List<NodeId> pastedNodeIds()`（execute 后有效），供 UI 粘贴后选中新节点。

### 5. cut = copy + CompositeCommand(RemoveNodeCommand × N + RemoveGroupCommand × M)
- 复用 TaskC 的 RemoveNodeCommand / RemoveGroupCommand，单一 CompositeCommand 作为一次撤销单元。
- RemoveNodeCommand 级联捕获连线；若两被删节点互联，连接只被先执行的那个命令捕获（另一个执行时连接已被级联删除），undo 逆序恢复，`addConnection` 对同一连接幂等——无重复/丢失。
- cut 不需要新命令类。

### 6. 连线忠实重建：携带 rule 引用，用 addConnection（绕过重校验）
- ConnectionSnapshot 记 `autoConverted` + `rule`（TypeConversionRule 引用，可空）。
- paste 用 `graph.addConnection`（TaskA 低级原语，不做类型校验、不做替换）原样恢复，**不调 `graph.connect`**。理由：连接在 copy 时已校验；同一会话类型系统不变；保持 autoConverted 标记与 rule 对象一致。
- 对比：若用 `graph.connect`，会重新查 TypeRegistry 派生 rule——registry 变更或规则移除会导致连接丢失或抛异常。addConnection 更忠实。

#### `SelectionCodec`
```java
public final class SelectionCodec {
    /** 从 graph 编码选中集合为自描述快照。nodes/groups 顺序保留（LinkedHashMap）。 */
    public static SelectionSnapshot encode(NodeGraph graph,
                                           Collection<NodeId> nodes,
                                           Collection<NodeGroupId> groups);

    /** 构造 PasteCommand（不执行）。offset 应用到节点与组位置。 */
    public static PasteCommand decode(SelectionSnapshot snapshot,
                                      NodeGraph graph,
                                      double offsetX, double offsetY);
}
```

**encode 算法**：
1. `nodeLocalId`：按 nodes 集合迭代序分配 0,1,2,...（LinkedHashMap<NodeId,Long>）。
2. `groupLocalId`：按 groups 集合迭代序分配 0,1,2,...。
3. `definitions`：遍历选中节点，`map.put(node.definition().id(), node.definition())`。
4. 节点条目：`localId`、`defId=node.definition().id()`、`header=node.header()`、`x/y`、`widgetValues`（遍历 `node.widgets()` 收 key→currentValue）、`localGroupId`（`node.groupId()` 非空且在 groupLocalId → 其本地 id；否则 null）。
5. 组条目：`localId`、`header`、`x/y/w/h/scale`。
6. 连线条目：遍历 `graph.connections()`，仅当 `fromNode` 与 `toNode` 均在 nodeLocalId → 记录（本地 id + 端口索引 + autoConverted + rule）。

#### `PasteCommand implements Command`
```java
public final class PasteCommand implements Command {
    public PasteCommand(NodeGraph graph, SelectionSnapshot snapshot, double offsetX, double offsetY);
    @Override public void execute();   // 懒捕获：首次分配+建图；redo 复用 saved 对象
    @Override public void undo();      // removeNode(级联) + removeGroup
    @Override public String description();  // "Paste"
    public List<NodeId> pastedNodeIds();    // execute 后有效
}
```

**execute（首次）**：
1. （构造期或 execute 初）校验：对每个节点条目，`defId` 必须能在 `snapshot.definitions()` 找到——否则抛 `IllegalArgumentException`（"Unknown definition: ..."），且**在任何变更前**完成全部解析（fail-fast，不半粘贴）。
2. 对每个组条目：`graph.createGroup(header, x+ox, y+oy, w, h)` → `setScale(scale)` → 保存；`groupIdMap[localGid]=group.id()`。
3. 对每个节点条目：`def=snapshot.definitions().get(defId)`；`node=graph.addNode(def, x+ox, y+oy)`；`node.setHeader(header)`；遍历 `node.widgets()` 按 key 用 `widgetValues` 覆盖 `setCurrentValue`（快照中缺失的 key 不动，保留 def 默认）；保存；`nodeIdMap[localId]=node.id()`。
4. 对每个节点条目：若 `localGroupId != null` → `graph.setNodeGroup(nodeIdMap[localId], groupIdMap[localGid])`。
5. 对每个连线条目：构造 `new Connection(nodeIdMap[fromLocal], fromOutput, nodeIdMap[toLocal], toInput, autoConverted, rule)`；`graph.addConnection(conn)`；保存。
6. `executed=true`。

**execute（redo）**：for group in savedGroups: `insertGroup`；for node in savedNodes: `insertNode`（groupId 已固化于对象）；for conn in savedConnections: `addConnection`。

**undo**：for node in savedNodes: `graph.removeNode(node.id())`（级联删连线）；for group in savedGroups: `graph.removeGroup(group.id())`。

#### `Clipboard`
```java
public final class Clipboard {
    public void copy(NodeGraph graph, Collection<NodeId> nodes, Collection<NodeGroupId> groups);
    public void cut(NodeGraph graph, Collection<NodeId> nodes, Collection<NodeGroupId> groups, UndoManager undo);
    /** 应用 PasteCommand 到 undo；快照为空时返回 null（不入栈）。返回已执行的 PasteCommand 供 UI 取 pastedNodeIds。 */
    public PasteCommand paste(NodeGraph graph, UndoManager undo, double offsetX, double offsetY);
    public SelectionSnapshot snapshot();   // 可空
    public boolean hasContent();
    public void clear();
}
```
- `copy`：`this.snapshot = SelectionCodec.encode(graph, nodes, groups)`（空选择→空快照，hasContent 仍 true）。
- `cut`：先 `copy`；再构造 `CompositeCommand("Cut", [RemoveNodeCommand × 每个选中节点, RemoveGroupCommand × 每个选中组])`；`undo.apply(composite)`。空选择→不入栈（直接返回）。
- `paste`：`if (snapshot==null) return null;` `PasteCommand cmd = SelectionCodec.decode(snapshot, graph, ox, oy);` `undo.apply(cmd);` `return cmd;`

## 不变量
1. SelectionSnapshot 不可变（record + copyOf）；definitions 覆盖所有节点条目的 defId。
2. encode 只包含两端均在选中集合的内部连线；外部连线被丢弃。
3. paste 产生的新 id 与原图任何现存 id 不冲突（addNode/createGroup 保证）。
4. PasteCommand 的 execute↔undo 可重复（undo→redo→…）且每次恢复到等价状态；saved 对象 id 跨循环稳定（insertNode/insertGroup 复用）。
5. paste 不修改原图既有对象（仅 addNode/createGroup 新增）。
6. cut 的 CompositeCommand 为单一撤销条目；成员 RemoveNodeCommand/RemoveGroupCommand 各自满足 TaskC 不变量。
7. 同一快照多次 paste 产生相互独立的新对象集（id 两两不同）。

## 异常行为
- `SelectionCodec.encode`：未知 NodeId/NodeGroupId（不在图中）→ `IllegalArgumentException`（透传 `graph.node`/`graph.group`）。
- `PasteCommand.execute`：defId 在 snapshot.definitions 找不到 → `IllegalArgumentException`（理论上 encode 自洽不会发生，防御性抛出）；**在变更前全部解析**。
- `graph.addConnection`（paste 内）endpoints 不存在（不应发生，节点已建）→ `IllegalArgumentException`；该 input 已被**另一**连接占据 → `IllegalStateException`。paste 重建的是 copy 时已校验的拓扑，正常不触发。
- `Clipboard.paste` 快照为空 → 返回 null，不抛、不入栈。
- `Clipboard.cut` 空选择 → 直接返回，不 apply 空 composite。

## 副作用
- copy：仅读图，填充 Clipboard 内部 snapshot。
- cut：填充 snapshot + 经 UndoManager apply 一个 CompositeCommand（修改图、触发监听器）。
- paste：经 UndoManager apply 一个 PasteCommand（修改图、触发监听器）。
- 无 IO、无磁盘持久化。

## 决策记录
1. **不引入 NodeDefinitionRegistry**：快照自带 definitions 映射使 TaskD 自闭合；避免范围蔓延。磁盘持久化时再加 registry。
2. **快照携带 def 引用**：内存剪贴板范围内 NodeDefinition（含 lambda）无法磁盘序列化，但可作进程内引用。defId 仍记入节点条目供自描述与未来序列化。
3. **本地 id 双命名空间**（节点 0..N、组 0..M）：简洁，连线引用节点本地 id，成员关系引用组本地 id，无歧义。
4. **PasteCommand 自洽**（非 CompositeCommand）：避开 AddNodeCommand execute 时分配 id 导致的构造期 id 预解析难题。复用 addNode/createGroup（首次）+ insertNode/insertGroup/addConnection（redo）的低级模式，与 AddNodeCommand 一致。
5. **不新增 NodeGraph API**：TaskA 的 addNode/createGroup/insertNode/insertGroup/addConnection/setNodeGroup/removeNode/removeGroup 已足够。
6. **连线用 addConnection（带 rule）而非 connect（重校验）**：忠实恢复 copy 时刻语义；不被会话期内 TypeRegistry 变更影响。
7. **cut 复用 RemoveNodeCommand/RemoveGroupCommand 的 CompositeCommand**：不造新命令；级联连线捕获 + addConnection 幂等保证无重复/丢失。
8. **header 与 widgetValues 全量记录**：捕获用户改名与参数调整，粘贴忠实还原。
9. **部分组选中→粘贴组含子集**：成员关系派生自 node.groupId，自然语义，无需特判。
