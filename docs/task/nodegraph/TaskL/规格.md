# TaskL 节点目录与添加（BUG 修复 + 3 设计补全）

## 背景
runClient 端到端测试发现 1 BUG + 3 设计补全需求。

## BUG：右键菜单 group 后无法再次弹出
- 根因：`groupSelection()`（NodeGraphWidget:352）执行后 `selection.clear()`。下次右键空白时 `openMenu()`（:388）因 selection 空 + clipboard 空 → items 空 → :416 `return` → 菜单不弹。
- 修复：随 L2，`openMenu` 即使无选择也提供"Add Node"项，items 必非空。

## L1 节点定义白名单 NodeDefinitionCatalog（纯 Java）
### 交付
- `api.def.NodeDefinitionCatalog`：`register(NodeDefinition)`(已存在 id→IllegalArgumentException)/`get(id)`(缺失抛)/`contains`/`all()`/`size`/`isEmpty`/`matching(query)->List`（header getString contains 忽略大小写，空=全部）/`withInputType(Type)`/`withOutputType(Type)`。
- `NodeGraph` 加可选 catalog 字段（默认 null）+ `catalog()`/`setCatalog()`。**addNode 不校验**（catalog 为 UI 过滤器，数据层开放，paste/程序化添加不受限）。
### 决策
- D1：catalog 是"建议白名单"非强制校验。NodeGraph 持有供 widget 查询；用户通过 UI 只能加 catalog 内的 def，但程序化/paste 不受限（符合"规定允许使用的节点集"=UI 视角的允许集）。

## L2 添加节点 UI AddNodeOverlay（客户端）
### 交付
- `client.widget.AddNodeOverlay`：持有 EditBox searchBox + 过滤列表 + 选中索引 + onPick 回调。
  - 构造 `(Font,int screenX,int screenY,List<NodeDefinition> initial,Consumer<NodeDefinition> onPick)`。
  - `render(g,mx,my)`：背景+EditBox.render+列表行（MAX_ROWS=8，hover/selected 高亮）。
  - `keyPressed(keyCode)->boolean`：EditBox 优先；Enter=确认选中+onPick+close；↑↓选；ESC=close。
  - `charTyped(char)->boolean`：EditBox。
  - `mouseClicked(mx,my)->boolean`：点列表行=确认；点 searchBox=focus；外部=close。
  - 内部 `refresh()`：据 searchBox 值过滤 + 重置 selectedIndex=0。
- `NodeGraphWidget`：加 addNodeOverlay 字段；`openAddNodeOverlay(screenX,screenY,worldX,worldY,Predicate<NodeDefinition> filter,Consumer<NodeDefinition> onPick)`；`addNodeOverlay()` getter；事件转发（mouseClicked/keyPressed/charTyped 优先 overlay）；render 顶层画 overlay。
- `openMenu` 加"Add Node"项（`graph.catalog()!=null`）→ openAddNodeOverlay（filter=null，onPick=addNode(def,wx,wy)）。**BUG 随此解决**。
- `NodeGraphScreen.keyPressed` 优先 canvas overlay；加 `charTyped` override 转发 canvas。

## L3 拖线到空白→类型过滤菜单
### 交付
- `NodeInteractionController.tryConnect`：drop 端口未命中（空白）→ `widget.openAddNodeOverlay`，typeFilter=源端口 type（拖输出→withInputType；拖输入→withOutputType），onPick=addNode+autoConnect。
- autoConnect：记录源(NodeId,portIndex,fromOutput)；新节点 addNode 后找首个类型精确匹配的对应端口（输出拖→新节点首个 input of 同 type；输入拖→新节点首个 output of 同 type）→ ConnectCommand 经 undo。找不到匹配端口则只加节点不连线。
- widget.openAddNodeOverlay 重载接收 autoConnect 参数。

## L4 DemoGraphFactory 注册 catalog + 验证
- create()：new NodeDefinitionCatalog，register constant/add/to_byte，graph.setCatalog(catalog)。
- runClient 端到端：右键空白→Add Node→搜索→放置；拖线到空白→过滤菜单→自动连线；右键 group 后再右键→菜单正常弹。

## 不变量
- catalog null 时 Add Node 项不出现（无来源）。
- AddNodeOverlay 开启时，鼠标/键盘事件被它独占（canvas 交互暂停）。
- 自动连线只精确类型匹配（不自动转换，避免歧义）。

- L1：TestNodeDefinitionCatalog（register/get/duplicate throws/matching 大小写+空 query/withInputType/withOutputType）。
- L2/L3：compileJava 通过（客户端无单测条件）。
- runClient 端到端确认交互。
