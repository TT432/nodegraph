# TaskE 反馈问题

本文件记录 TaskE 执行过程中发现的规格文档（`docs/task/nodegraph/TaskE/规格.md`）问题。

## 1. setScale 异常条款与验证清单冲突

- **规格异常条款（第 125 行）**：`Viewport.zoom/setScale`：`factor<=0` 或 `scale<=0` → `IllegalArgumentException`。
  读作：zoom 的 factor<=0 抛异常；setScale 的 scale<=0 抛异常。
- **验证清单第 7 项（第 156 行）**：`setScaleClamps`：setScale(100,...) → MAX_SCALE；**setScale(0,...) → MIN_SCALE**。
  setScale(0) 应限幅到 MIN_SCALE，不抛异常。

两者冲突：setScale(0,...) 既要求"抛异常"又要求"→ MIN_SCALE"。

- **处理**：以**验证清单**（可执行测试）为权威，setScale 对任意输入（含 <=0）一律限幅，不抛异常。
  仅 zoom 的 factor<=0 抛异常（清单第 8 项 + 异常条款一致）。已在 `Viewport.setScale` 源码注释记录。

## 2. 滚轮平移方向：公式与文字解释冲突

- **规格 D7 第 61 行**：无修饰键 → `panY -= delta * SCROLL_SPEED`。
- **同行括号解释**：向上滚→内容上移即看下方内容。

按公式推导（`screenY = (worldY - panY) * scale + originY`）：delta>0（向上滚）使 panY 减小，
固定 worldY 的点 screenY 增大 → 内容在屏幕上下移 → 看到的是上方（更小 worldY）的内容，
与括号"看下方内容"相反。

- **处理**：本环境无 runClient 实测条件，按**公式**实现（`panY -= delta * SCROLL_SPEED`，即
  `viewport.pan(0, delta * SCROLL_SPEED * scale)`）。Shift 横向同理 `panX -= delta * SCROLL_SPEED`。
  视觉方向是否合用户习惯，留待用户 runClient 时确认；如方向相反，翻转 mouseScrolled 中的符号即可。

## 3.（非规格问题，实现备忘）修饰键 API 位置

规格未指定修饰键检测 API。`hasControlDown()`/`hasShiftDown()` 在本映射（parchment 1.20.1）中是
`net.minecraft.client.gui.screens.Screen` 的 **public static** 方法，**不在** `GuiEventListener`。
NodeGraphWidget 用 `Screen.hasControlDown()` / `Screen.hasShiftDown()`。
