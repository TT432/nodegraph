# TaskE 画布与 Widget 基础（客户端）

## 目标

为节点图编辑器构建客户端 GUI 的**画布骨架**：一个可嵌入任意 `Screen` 的 `NodeGraphWidget`，负责
- 持有 `NodeGraph` 数据模型与视口状态（平移/缩放）；
- 世界↔屏幕坐标变换；
- 网格背景渲染；
- 平移（中键拖 / 左键拖空白 / 鼠标滚轮）；
- 缩放（Ctrl+滚轮）；
- 滚动（滚轮即视口平移）。

本任务**不含**节点渲染（TaskF）、连线渲染（TaskG）、节点交互（TaskH）、选择/菜单（TaskI）、节点组交互（TaskJ）。但本任务的坐标变换、视口、事件状态机是后续所有客户端 Task 的公共基础。

## 前置

- TaskA 数据模型：`NodeGraph`、`Node`、`NodeGroup`（纯数据，公共侧类，客户端可引用）。
- 复用 TaskA 的 `Component`（`net.minecraft.network.chat.Component`）。
- 构建配置复用 TaskA 的 JUnit5 + `testCompileClasspath/testRuntimeClasspath.extendsFrom(modDevCompileDependencies)`。

## 关键设计决策

### D1：视口状态归 Widget，不进 NodeGraph
NodeGraph 是纯数据（TaskA 假设已确认）。平移/缩放属于"观察方式"，归客户端视口。**`Viewport` 抽为纯 Java 类**（无 MC 依赖），便于 JUnit 验证坐标数学。NodeGraphWidget 持有一个 `Viewport` 实例。

### D2：世界/屏幕坐标约定
- **世界坐标** = NodeGraph 中 `node.x()/node.y()` 所在坐标系（双精度浮点，逻辑像素，不受缩放影响）。
- **屏幕坐标** = widget 内的 GUI 像素（int/double，受 widget 在 Screen 中的位置 + 视口平移 + 缩放影响）。
- 变换公式（`scale>0`）：
  - `screenX = (worldX - panX) * scale + widgetX0`
  - `screenY = (worldY - panY) * scale + widgetY0`
  - `worldX = (screenX - widgetX0) / scale + panX`
  - `worldY = (screenY - widgetY0) / scale + panY`
- `panX/panY` 是世界坐标系中"视口左上角对应的世界点"。`widgetX0/widgetY0` 是 widget 在 Screen 中的左上屏幕坐标（= `getX()/getY()`）。
- **缩放围绕锚点**（光标处）：缩放前后该世界点保持在同一屏幕坐标。即缩放时调整 panX/panY 使得锚点世界坐标经新 scale 后屏幕位置不变。

### D3：scale 范围与限幅
`scale ∈ [MIN_SCALE=0.1, MAX_SCALE=4.0]`（覆盖 10%–400%）。`zoom(factor)` 后对 scale 限幅；若 factor 使其越界则 clamp，仍调整 pan 以锚点为准（用 clamp 后的 scale）。

### D4：事件派发链约束与中键拖动方案（关键）
经核实 MC 1.20.1 源码（`ContainerEventHandler`/`AbstractWidget`/`Screen`）：
- `ContainerEventHandler.mouseDragged` 默认实现：`getFocused()!=null && isDragging() && button==0` 才派发 → **中键(2)/右键(1)拖动事件默认不到达子 widget**。
- `ContainerEventHandler.mouseClicked`：子元素返回 true 时 `setFocused`，但 `setDragging(true)` 仅当 `button==0`。
- `AbstractWidget.mouseClicked` 受 `isValidClickButton()`（默认 `button==0`）+ `clicked()`（光标在 widget 边界内）双重约束。
- `mouseScrolled(double,double,double)`：经 `getChildAt` 路由到光标下子元素，widget 在光标下即可收到。**1.20.1 为单 `delta` 参数**。

**方案**：TaskE 交付
1. `NodeGraphWidget`（extends `AbstractWidget`）—— 可嵌入任意 Screen；override `mouseClicked/mouseReleased/mouseDragged/mouseScrolled/mouseMoved/isValidClickButton/updateWidgetNarration/renderWidget`。内部维护 `InteractionState`（IDLE/PANNING）。对于默认派发链可达的事件（左键拖 button0、滚轮、mouseClicked/mouseReleased 全按钮），widget 直接处理。对于中键拖动，依赖宿主 Screen 转发 `mouseDragged(button=2)` 或 `mouseMoved`。
2. `NodeGraphScreen`（extends `Screen`）—— **便利宿主**，override `mouseClicked/mouseReleased/mouseDragged/mouseScrolled/mouseMoved`，把事件显式派发给其持有的 `NodeGraphWidget`（无论按钮号），保证中键拖动等在默认集成下可用。供 TaskK 直接使用；其他想嵌入 widget 的 Screen 必须按同样契约转发（文档记录）。

### D5：网格渲染
- 网格在世界坐标系中，间距 `GRID_SIZE=20` 世界单位（缩放后视觉间距 = `GRID_SIZE * scale` 屏幕像素）。
- 仅渲染视口可见范围内的网格线（避免渲染数千条线）。
- 网格颜色：暗灰 `0xFF2B2B2B`；原点轴（x=0,y=0）稍亮 `0xFF444444`。
- 背景填 `0xFF1A1A1A`（深灰，区分屏幕其他 UI）。

### D6：scissor 裁剪
渲染节点内容前 `guiGraphics.enableScissor(x0,y0,x1,y1)`（widget 边界），渲染后 `disableScissor()`，确保节点/网格不溢出 widget 区域。TaskE 阶段裁剪网格即可（无节点）。

### D7：滚轮语义
- **无修饰键**：`panY -= delta * SCROLL_SPEED`（垂直滚动，向上滚→内容上移即看下方内容）。
- **Shift**：`panX -= delta * SCROLL_SPEED`（横向滚动）。
- **Ctrl**：`zoom`，`scale *= pow(ZOOM_FACTOR, delta)`，锚点为光标。`ZOOM_FACTOR=1.15`，`SCROLL_SPEED=16`。
- `delta>0` = 向上滚（标准）。

## 交付物

### 纯 Java 层（可 JUnit 测试）
包 `com.nodegraph.nodegraph.client.viewport`：
- `Viewport`（final class，可变）：字段 `double panX, panY, scale`（包级可见 setter for 测试）；构造默认 `panX=panY=0, scale=1`。
  - `double screenToWorldX(double sx, double originX)` / `screenToWorldY(sy, originY)`
  - `double worldToScreenX(double wx, double originX)` / `worldToScreenY(wy, double originY)`
  - `void pan(double dxScreen, double dyScreen)`：`panX -= dxScreen/scale; panY -= dyScreen/scale`（屏幕右拖→看左边内容，故减）。
  - `void zoom(double factor, double anchorScreenX, double anchorScreenY, double originX, double originY)`：先算锚点世界坐标，scale*=factor 后 clamp，再以新 scale 反算 pan 使锚点屏幕坐标不变。
  - `void setScale(double s, anchor...)`：直接设 scale（clamp）+ 锚点保持。
  - `double scale()` / `double panX()` / `double panY()`（getter）。
  - 常量 `MIN_SCALE=0.1`、`MAX_SCALE=4.0`。

### 客户端层（不可 JUnit 测试，compileJava 验证）
包 `com.nodegraph.nodegraph.client.widget`：
- `NodeGraphWidget`（extends `AbstractWidget`）：
  - 构造 `(int x, int y, int width, int height, NodeGraph graph)`；super 调用传 `Component.empty()` 作 message。
  - 字段：`NodeGraph graph`、`Viewport viewport`、`InteractionState state`（enum IDLE/PANNING）、`int panButton`、`double lastMouseX, lastMouseY`。
  - `NodeGraph graph()` / `Viewport viewport()`。
  - `protected void renderWidget(GuiGraphics g, int mouseX, int mouseY, float partialTick)`：填背景 → enableScissor → 画网格 → disableScissor。
  - `@Override public boolean isValidClickButton(int b)`：return `b==0||b==1||b==2`。
  - `@Override public boolean mouseClicked(double mx, double my, int button)`：仅当 `clicked(mx,my)`（在 widget 边界内）；button∈{0,1,2} → 进入 PANNING（panButton=button，记录 lastX/lastY）；返回 true 消费；否则 false。
  - `@Override public boolean mouseReleased(double mx, double my, int button)`：若 state==PANNING → IDLE，返回 true；否则 false。
  - `@Override public boolean mouseDragged(double mx, double my, int button, double dx, double dy)`：若 state==PANNING && button==panButton → `viewport.pan(dx,dy)`，更新 lastX/lastY，返回 true；否则 false。
  - `@Override public void mouseMoved(double mx, double my)`：若 state==PANNING（中键场景，宿主经 mouseMoved 转发）→ 计算 dx=mx-lastX, dy=my-lastY，viewport.pan，更新 last；否则 no-op。
  - `@Override public boolean mouseScrolled(double mx, double my, double delta)`：在 widget 边界内 → 按 D7 语义（Shift/Ctrl）调整 viewport，返回 true；否则 false。
  - `@Override protected void updateWidgetNarration(NarrationElementOutput o)`：默认 narration（`defaultButtonNarrationText(o)`）。
  - `protected void renderGrid(GuiGraphics g)`：根据 viewport 计算可见世界范围 → 画网格线（fill 矩形宽/高 1px）。
  - 私有 `boolean clicked(double mx, double my)`：复用边界判定 `mx>=getX() && my>=getY() && mx<getX()+width && my<getY()+height`。

- `NodeGraphScreen`（extends `Screen`）：
  - 构造 `(Component title, NodeGraph graph)`。
  - 字段 `NodeGraphWidget canvas`。
  - `@Override protected void init()`：`canvas = addRenderableWidget(new NodeGraphWidget(0,0,width,height,graph))`。
  - `@Override public boolean mouseClicked(double mx,double my,int b)`：转发到 canvas（先 super，若未消费再 canvas.mouseClicked）——实现：`return super.mouseClicked(mx,my,b) || canvas.mouseClicked(mx,my,b)`。**简化为直接 canvas 路由**：`return canvas.mouseClicked(mx,my,b)`（canvas 占满屏）。
  - 同样 `mouseReleased/mouseDragged/mouseScrolled` 转发 canvas；`mouseMoved` 转发 `canvas.mouseMoved`。
  - `@Override public boolean mouseScrolled(...)`：return `canvas.mouseScrolled(mx,my,delta)`。
  - `@Override public void render(...)`：super（画 renderables→canvas）+ 可选 HUD（缩放百分比文字，右下角）。
  - `@Override public boolean isPauseScreen()`：return false（单机集成不暂停）。
  - `NodeGraphWidget canvas()` getter。

### 包结构
```
com.nodegraph.nodegraph.client.viewport.Viewport           (纯Java, JUnit)
com.nodegraph.nodegraph.client.widget.NodeGraphWidget       (客户端)
com.nodegraph.nodegraph.client.widget.NodeGraphScreen       (客户端)
```

## 不变量

1. `Viewport.scale ∈ [MIN_SCALE, MAX_SCALE]`，任何 pan/zoom/setScale 后恒成立。
2. `zoom(factor, anchor)` 调用前后，锚点屏幕坐标对应的世界坐标不变（数值精度内）。
3. `worldToScreenX(screenToWorldX(sx,ox),ox) == sx`（数值精度内），Y 同理；`pan` 改变不影响此恒等。
4. NodeGraphWidget 在 PANNING 状态下，mouseReleased(button==panButton) 必置 IDLE。
5. NodeGraphWidget 不修改 NodeGraph（视口为只读观察；节点编辑交互在 TaskH+）。
6. 所有鼠标事件处理仅在光标位于 widget 边界内时消费（边界外返回 false，允许事件冒泡）。

## 异常行为

- `Viewport.zoom/setScale`：`factor<=0` 或 `scale<=0` → `IllegalArgumentException`。
- `NodeGraphWidget` 构造：`graph==null` → `NullPointerException`（Objects.requireNonNull）。
- `NodeGraphScreen` 构造：`graph==null` → `NullPointerException`。

## 副作用

- 纯 Java `Viewport`：仅改变自身字段。
- `NodeGraphWidget`：改变自身 viewport 字段；不改 NodeGraph；不改 Minecraft 全局状态。
- 渲染调用 `GuiGraphics.fill/enableScissor/disableScissor`（标准 GUI 副作用）。
- `NodeGraphScreen.init` 替换 `canvas` 字段（Screen 重建时）。

## 决策记录

1. **Viewport 纯 Java**：使坐标变换数学可 JUnit 验证（客户端代码无法 runClient 实测，纯 Java 子集可测）。
2. **缩放围绕光标锚点**：行业惯例（Blender/PS/网页地图），缩放时鼠标下的内容保持在鼠标下。
3. **pan 方向**：屏幕拖动向量 `(dx,dy)` → 世界内容跟随鼠标方向移动，故 `panX -= dx/scale`（拖右→内容右移→看左）。
4. **中键拖动经 NodeGraphScreen 转发**：MC 默认派发链不路由 button!=0 的拖动（D4），便利宿主解决；widget 自身对默认可达事件（左键拖、滚轮）自洽。
5. **网格世界空间而非屏幕空间**：缩放时网格自然变密/疏，符合"画布"语义。
6. **网格密度自适应**（后续优化点）：scale 过小时网格太密。TaskE 范围内不做密度自适应，但预留 `GRID_SIZE` 常量便于后续按 scale 倍增。
7. **isPauseScreen=false**：编辑器界面不应暂停游戏（与创造栏一致）。
8. **不实现滚动条 UI**：用户需求"横向/竖向轴滚动"经滚轮平移满足；可见滚动条属视觉增强，延后。

## 验证清单（JUnit 5，纯 Java Viewport）

`src/test/java/com/nodegraph/nodegraph/client/viewport/TestViewport.java`：
1. `identityTransform`：scale=1,pan=0 → worldToScreen 与 screenToWorld 互逆。
2. `panMovesWorldOrigin`：pan(10,0) 后 screenToWorldX(0,0) == -10/scale。
3. `scaleAffectsScreenSize`：scale=2 时 worldToScreenX(10,0)==20（1 世界单位=2 屏幕像素）。
4. `zoomKeepsAnchorWorldPoint`：zoom(2, ax, ay, originX, originY) 前后，`screenToWorldX(ax,originX)` 不变。
5. `scaleClampedToMax`：从 scale=3 zoom(factor=10) → scale==MAX_SCALE。
6. `scaleClampedToMin`：从 scale=0.2 zoom(factor=0.01) → scale==MIN_SCALE。
7. `setScaleClamps`：setScale(100,...) → scale==MAX_SCALE；setScale(0,...) → MIN_SCALE。
8. `zoomWithNonPositiveFactorThrows`：zoom(0,...) 与 zoom(-1,...) 抛 IllegalArgumentException。
9. `roundTripAfterPanAndZoom`：组合 pan+zoom 后 screenToWorld(worldToScreen(x))==x（多组采样）。
10. `defaultViewportIsIdentity`：新 Viewport() scale==1,panX==0,panY==0。

客户端代码（Widget/Screen）验证标准：**compileJava 通过**（无 runClient 实测条件）；结构正确性由代码审查保证，视觉效果由用户 runClient 确认。
