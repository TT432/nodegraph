# NodeGraph 规划文档

## 1. 任务目标
为 Minecraft（Forge 1.20.1）实现一个**可复用的节点图编辑器库 + 求值引擎**。库对外提供：
- 一个可插入任意 `Screen` 的 `NodeGraphWidget`（UI 组件）。
- 类型系统、节点 / 连线 / 节点组的数据模型。
- Blender 风格的惰性求值引擎。
- 完整交互：拖拽、连线、框选、右键菜单、复制/剪切/粘贴、撤销/重做、节点组。

入口：注册一个物品，右键打开内置 `NodeGraphScreen`；附带演示节点定义。

## 2. 环境基线
- Forge 1.20.1（forge 47.4.20），ModDevGradle `legacyforge` 插件（2.0.141），Java 17。
- `mod_id = nodegraph`，`mod_group_id = com.nodegraph.nodegraph`。
- 当前为 MDK 模板，含示例 Block / Item / CreativeTab（将在 TaskK 清理为正式内容）。
- MC 文本组件 = `net.minecraft.network.chat.Component`；GUI 渲染 = `net.minecraft.client.gui.GuiGraphics`；屏幕 = `net.minecraft.client.gui.screens.Screen`；组件基类 = `net.minecraft.client.gui.components.AbstractWidget`。

## 3. 架构决策
1. **模型 / 视图分离**
   - 纯 Java 层（数据模型、求值引擎、命令系统、剪贴板）**不依赖 MC 客户端**，可用 JUnit 单测。
   - MC 客户端层（Widget、渲染、交互）构建在纯 Java 层之上。
2. **单组件自渲染**：`NodeGraphWidget extends AbstractWidget`，自身负责全部渲染与事件处理（不挂子 widget 列表），通过 `addRenderableWidget` 加入任意 `Screen`。
3. **坐标系**：世界坐标 (world) 与屏幕坐标 (screen)，经 `panOffset + scale` 互转；支持平移、缩放、轴向滚动。
4. **连线语义**：输出端口持有出向 `Connection` 列表（可扇出）；输入端口最多一条入向 `Connection`（单一源）；向已连接的输入再连新线 → **替换**原连线。
5. **类型与转换**：`Type` 内在为字符串标识；`TypeRegistry` 注册类型并确定颜色（按 id 稳定哈希或显式注册）；`TypeConversionRule` 注册 `A→B` 转换；类型不同但有匹配规则时允许连接，标记为**自动转换**并显示警告（虚线/警告色）。
6. **求值引擎（参考 Blender）**：惰性求值。求一个输出端口 → 递归解析该节点各输入来源（连线源输出 *或* 体部输入组件的当前值）→ 调用节点求值函数；环检测报错；单次求值 pass 内缓存结果避免重复计算。
7. **撤销/重做**：Command 模式 + `UndoManager`；所有变更经命令；`Ctrl+Z` / `Ctrl+Shift+Z`。
8. **剪贴板**：选择集（节点 + 其内部连线）序列化；粘贴时重映射 id。
9. **入口**：`NodeEditorItem` 右键打开 `NodeGraphScreen`。

## 4. 记录的假设与选择
> 按 AGENTS.md，模棱两可之处在此记录；执行期若遇新选择，补充到对应子任务文档的"决策记录"。
- 连接扇出/单源：采用标准约定（用户已确认：输出可扇出，输入单一源）。
- 向已连接输入再连线 = 替换（同 Blueprint 约定）。
- 输入端口未连线时，其值由体部"输入组件"（文本/滑块/按钮组/下拉框）提供——Blender 惯例。
- **客户端代码在本环境无法 `runClient` 实测**，验证标准 = `compileJava` 通过 + 结构正确性；视觉效果由用户运行 `runClient` 确认。纯 Java 层用 JUnit 5 实测。

## 5. 包结构（规划）
```
com.nodegraph.nodegraph
├─ api
│  ├─ type       Type, TypeRegistry, TypeConversionRule
│  ├─ model      TypedValue, Port, Node, NodeGroup, NodeGraph, Connection, InputWidget
│  ├─ def        NodeDefinition, NodeFunction, InputWidgetKind
│  ├─ eval       Evaluator, EvaluationContext
│  ├─ command    Command, UndoManager, 各具体命令
│  └─ clipboard  Clipboard, SelectionCodec
├─ client
│  ├─ ui         NodeGraphWidget, 渲染器, 交互状态机, 坐标变换
│  └─ screen     NodeGraphScreen
├─ common
│  └─ item       NodeEditorItem
└─ data          内置类型、演示节点定义
```

## 6. 子任务划分
每个子任务**可独立验证、互相孤立**。

| 编号 | 名称 | 层 | 验证 |
|------|------|----|------|
| TaskA | 类型系统与数据模型 | 纯 Java | JUnit |
| TaskB | 求值引擎 | 纯 Java | JUnit |
| TaskC | 命令系统与撤销/重做 | 纯 Java | JUnit |
| TaskD | 剪贴板与序列化 | 纯 Java | JUnit |
| TaskE | 画布与 Widget 基础（坐标变换/网格/平移/缩放/滚动） | 客户端 | compileJava + 结构 |
| TaskF | 节点渲染（头/体/组件/端口、类型配色、标签、tooltip） | 客户端 | compileJava + 结构 |
| TaskG | 连线渲染（贝塞尔、类型配色、自动转换警告、预览线） | 客户端 | compileJava + 结构 |
| TaskH | 节点交互（拖头移动、拖端口连线、类型校验、tooltip） | 客户端 | compileJava + 结构 |
| TaskI | 选择与右键菜单（框选、菜单、快捷键、接入命令+剪贴板） | 客户端 | compileJava + 结构 |
| TaskJ | 节点组（线框、头拖动、拖入/拖出成员、缩放） | 客户端 | compileJava + 结构 |
| TaskK | 入口与演示（物品开屏、演示节点、清理示例、端到端） | 集成 | compileJava + 端到端 |

## 7. 验证策略
- 纯 Java 层：JUnit 5。覆盖：数据模型约束（扇出/单源/替换）、引擎正确性与环检测、命令的执行/撤销/重做、剪贴板 id 重映射。
- 客户端层：`compileJava` 通过 + 结构审查；视觉效果由用户 `runClient` 确认。

## 8. 依赖与顺序
TaskA → TaskB（依赖模型）。TaskC/D 可与 B 并行但依赖 A。TaskE~J 依次构建同一 Widget，E 先建骨架与交互状态机，后续填充。TaskK 最后集成。
