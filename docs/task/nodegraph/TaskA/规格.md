# TaskA 类型系统与数据模型

## 目标
建立节点库的纯 Java 数据层：类型系统 + 图数据模型 + 连接约束。不依赖 MC 客户端渲染类（仅引用 `net.minecraft.network.chat.Component` 与 `ResourceLocation`，二者均为公共侧可用）。

## 前置条件
- Forge 1.20.1 / Java 17 工程可编译。
- 无（TaskA 是基础）。

## 范围与交付物
包 `com.nodegraph.nodegraph.api.*`：

### api.type
- `Type`：不可变。字段 `String id`、`int color`(ARGB)。相等性按 `id`。`color()` getter。
- `TypeConversionRule`：不可变。`Type from`、`Type to`、`Function<Object,Object> converter`。`apply(Object)`。
- `TypeRegistry`：
  - `register(id, color) -> Type`
  - `get(id) -> Type`（缺失抛 `IllegalArgumentException`）
  - `registerConversion(from, to, fn) -> TypeConversionRule`
  - `conversion(from, to) -> Optional<TypeConversionRule>`
  - `canConvert(from, to) -> boolean`：`from.equals(to)` 为恒等可转，或存在规则。
  - 颜色在 register 时指定。

### api.model
- `TypedValue`：不可变。`Component name`、`Type type`、`Component description`。
- `Port`：不可变。`String key`（求值用稳定标识）+ `TypedValue value`。便捷 `type()/name()/description()`。
- `InputWidgetKind`：枚举 `TEXT, SLIDER, BUTTON_GROUP, DROPDOWN`。
- `InputWidget`：`key`、`TypedValue value`、`kind`、可变 `Object currentValue`（节点本地参数值）。
- `Node`：`NodeId id`、`NodeDefinition definition`、可变 `Component header`、可变 `double x,y`、`List<InputWidget> widgets`、`List<Port> inputs`、`List<Port> outputs`、可变 `NodeGroupId groupId`(可空)。
- `NodeGroup`：`NodeGroupId id`、可变 `Component header`、可变 `double x,y,w,h`、可变 `double scale`(默认 1.0)。**成员关系不在 group 存储**——以 `Node.groupId` 为唯一真源；`NodeGraph.members(group)` 派生。
- `Connection`：不可变关系。`NodeId fromNode`、`int fromOutput`、`NodeId toNode`、`int toInput`、`boolean autoConverted`、`TypeConversionRule rule`(可空)。
- `NodeId` / `NodeGroupId`：值类型，封装 `long`，互不兼容（防混用）。

### api.def
- `NodeDefinition`：`ResourceLocation id`、`Component header`、`List<InputWidgetSpec> widgets`、`List<PortSpec> inputs`、`List<PortSpec> outputs`、`NodeFunction function`。
- `InputWidgetSpec`：`key, TypedValue, kind, Object defaultValue`。
- `PortSpec`：`key, TypedValue`。
- `NodeFunction`（函数式接口）：`Map<String,Object> evaluate(Map<String,Object> inputValues, Map<String,Object> widgetValues)`，返回输出键→值。

### api.model.NodeGraph（容器，纯数据）
- 节点：`addNode(definition, x, y) -> Node`（分配 NodeId，按定义实例化 widgets/inputs/outputs，widget 初值取 spec.defaultValue）。`removeNode(id)`（连带删除其全部 connection；清理 groupId 引用——节点本身被删）。`node(id)`、`nodes()`。
- 组：`createGroup(header, x, y, w, h) -> NodeGroup`、`removeGroup(id)`（成员节点的 groupId 置 null）、`members(group) -> List<Node>`（按 node.groupId 派生）、`setNodeGroup(nodeId, groupId)`。
- 连接（核心约束）：
  - `canConnect(fromNode, fromOut, toNode, in) -> ConnectResult`，其中 `ConnectResult` 枚举 `OK / AUTO_CONVERTED / INCOMPATIBLE`。
  - `connect(...) -> Connection`：校验 → **输入单一源**：若该输入已有连接，先移除（替换）→ 创建 Connection，登记。
  - `disconnect(Connection)`。
  - `inputConnection(node, in) -> Optional<Connection>`（单一源查询）。
  - `outputsOf(node, out) -> List<Connection>`（扇出查询）。
  - `connections()`。
- 内部 id 计数器分配 NodeId / NodeGroupId。

## 不变量
1. 每个 `NodeId` 唯一；删除后不复用。
2. 任一输入端口至多一条入向 Connection（单一源）。
3. 一个输出端口可有多条出向 Connection（扇出）。
4. Connection 端口索引必须对节点有效；引用已删节点的 Connection 不存在。
5. `Node.graphId` 与某现存组 id 一致，或为 null。
6. 仅当 `fromOut.type` 与 `in.type` 恒等或存在注册转换时 `canConnect` 非 INCOMPATIBLE。

## 异常行为
- `TypeRegistry.get(id)` 缺失 → `IllegalArgumentException`。
- `NodeGraph.node(id)` 缺失 → `IllegalArgumentException`。
- `connect` 类型不兼容 → `IllegalArgumentException`。
- `connect` 跨不存在的节点/端口 → `IllegalArgumentException`。

## 副作用
- 仅内存对象图；无 IO、无持久化。

## 决策记录
- 端口以 `(nodeId, index)` 寻址而非对象引用：避免 Port↔Connection 循环引用，序列化（TaskD）零成本。
- 成员关系单一真源 = `Node.groupId`；避免双向同步 bug。
- `Node` 头部可变（允许后续自定义标题）；ports/widgets 不可变（来自定义）。
- 视口状态（pan/scale）**不**放入 NodeGraph——属视图层，归 Widget（TaskE）。NodeGraph 为纯数据。
- 环检测不在数据层做：允许任意结构合法连接，求值期（TaskB）报环。

## 验证（JUnit 5）
`src/test/java/com/nodegraph/nodegraph/api/`：
1. `TestTypeRegistry`：注册、查询、缺失抛异常、canConvert 恒等/规则/不存在。
2. `TestNodeGraphConstraints`：
   - 扇出：同一输出连两个输入 → 两条 connection。
   - 单一源：同输入连第二次 → 替换，`inputConnection` 仅返回新的。
   - 类型校验：同类型 OK；不同但有转换 AUTO_CONVERTED；不同无转换 connect 抛异常。
   - removeNode 连带删除其连接。
   - 组成员派生与 setNodeGroup。
3. `TestNodeId`：NodeId 与 NodeGroupId 类型隔离（编译期已保证，单测验证相等性）。
